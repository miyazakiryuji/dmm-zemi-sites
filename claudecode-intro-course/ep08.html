EP08
Claude Codeの基本操作

デスクトップアプリの画面を一周しながら、よく使うボタンと「指示→生成→確認」の流れを体で覚える回です。

Claude Codeを起動すると、ボタンやメニューがたくさん並んでいて少し身構えてしまうかもしれません。でも大丈夫です。最初から全部を覚える必要はなく、よく使うものから少しずつ慣れていけば十分です。この回では、画面に並ぶ主要な機能をざっと見渡したうえで、実際に小さなファイルを作ってもらう簡単なハンズオンに挑戦します。

Learn

この回の学習ノート

1まずは画面を一周する（全部は覚えなくていい）

Claudeデスクトップアプリの「Claude」タブを開くと、Claude Codeを使える状態になります。画面にはいろいろなボタンが並んでいますが、一気に頭に入れようとしなくて大丈夫です。手を動かしているうちに、よく触る機能から自然と覚えていきます。

  • 新しい会話を始める：いまから取り組みたいタスクのために、まっさらなセッションを開始できます。
  • 履歴：過去のやりとりが蓄積され、あとから見返せます。
  • 入力欄と送信ボタン：AIへの指示（プロンプト）をここに書いて送ります。
  • ダッシュボード：使用状況などの全体像を確認できる領域です。
Claude Codeは新機能の追加や表示の変更が多く、ボタンの位置や呼び名が入れ替わることもあります。視聴のタイミングで画面が少し違っていても慌てず、似た役割の場所を探してみてください。

2実行場所と作業フォルダを決める

Claude Codeはファイルを読んだり書いたりして作業します。そのとき「どこで実行するか」と「どのフォルダを対象にするか」を最初に決めておくと、迷わずに済みます。

  • 実行場所：生成する環境はいくつか選べますが、最初は「ローカル」（自分のPC上）のままで問題ありません。他の環境は事前の準備に少し手間がかかるので、まずはローカルで慣れていきましょう。
  • 作業フォルダ：「フォルダを開く」から、自分のPC内の好きなフォルダを指定します。たとえばデスクトップに「テスト」フォルダを作っておき、それを選ぶといった使い方です。
作業用のフォルダは、練習で作ったファイルが散らからないように、専用の空フォルダを1つ用意しておくのがおすすめです。名前は何でも構いません。

3「編集を承認」モードでツール実行の確認レベルを決める

Claude Codeはファイルの作成や編集など、たくさんの「ツール」を呼び出して動きます。それぞれの実行について、本当に進めてよいかを人間側にどこまで尋ねるか。そのレベル感を切り替えられるのが「編集を承認」のメニューです。複数のモードが用意されているので、まずは「こういう段階があるんだな」と把握しておけば十分です。

「許可をバイパス」を選ぶと、どんなツールの実行も人間に尋ねずそのまま進むようになります。場合によっては意図しないファイルを消してしまうこともあるため、慣れないうちは有効にしないのが無難です。状況を見極められる上級者向けの設定だと考えてください。

4モデルと思考レベル、使用量ゲージ

ここはぜひ覚えておきたいポイントです。タスクの重さに合わせて、AIモデルと「どれくらい考えさせるか」を選べます。

  • モデル選択：基本はバランス型のSonnetでOK。込み入った重いタスクを任せたいときはOpus、簡単な計算や一言で片づくやりとりならさらに軽快なHaiku、というように使い分けます。
  • 思考レベル（低／中）：Claudeは自分で思考するように訓練された推論モデルで、その考える深さを調整できます。速さを優先したいなら「低」、もう少しじっくり考えてほしいときは「中」。最初はこの2つを場面で切り替えれば十分です。
  • 使用量ゲージ：利用制限に対して自分がどれだけ消費したかを確認できます。5時間ごとに使える量の枠が決まっていて、そこを使い切ると次の枠が来るまでClaude Codeが動かせなくなります。残量はこまめに気にしておきましょう。
迷ったら「Sonnet＋思考レベルは低か中」から始めるのがおすすめ。手応えを見ながら、重い作業のときだけOpusに上げるくらいの感覚で十分です。

5ハンズオン：ファイルを作って、内蔵ビューアで確認する

画面を一周したら、実際に指示を出してみましょう。ここでは2つの小さな指示を通して、「プロンプトを送る→生成される→その場で確認する」という流れを体験します。設定はSonnet・思考レベルは低・編集の承認モードのまま進めます。

  • 1つ目：テキストファイル「hello.txt」を作り、その中に自己紹介を3行で書いてもらうよう指示します。名前・趣味・好きなツールなど、中身は自由でOK。ごく短いファイルなので一瞬で出来上がります。
  • 確認：「View」を開き「ファイル」を選ぶと、アプリに組み込まれたフォルダ表示が使えます。生成された hello.txt をクリックすれば中身まで読め、アプリの中だけで一通り確認できます。
  • 2つ目：続けて、半角の「@」で先ほどのファイルを指定し、その内容をもとに自分の名前を大きく出すだけの簡単なHTMLファイルを作ってブラウザで開いてもらいます。HTMLは画面に何かを表示するためのマークアップ言語で、内蔵のブラウザプレビューでそのまま見た目を確認できます。
便利な操作を2つ。Shift＋Enterを押すと入力欄で改行できます。さらに行頭に半角の「@」を打つとファイルを選べて、「このファイルを見て」と参照させられます。@でのファイル指定はとてもよく使うので、ぜひ覚えておきましょう。

このように、短い日本語の指示を送るだけで、テキストファイルを作ったり、その内容を読み取ってHTMLを生成したりできます。まずは色々な指示を試して、画面の操作に慣れていきましょう。

Key Points

この回のポイント

  • 画面の機能は全部覚えなくてよい。よく使うものから慣れればOK（アップデートで配置が変わる点にも注意）。
  • 実行場所はまずローカルのまま。作業フォルダは「フォルダを開く」で自分のPCの任意フォルダを指定する。
  • 「編集を承認」でツール実行の確認レベルを設定できる。「許可をバイパス」は危険なので初心者は使わない。
  • モデルは基本Sonnet、複雑ならOpus、軽い処理はHaiku。思考レベルは低／中で調整する。
  • 使用量ゲージで5時間ごとの枠の残量を確認できる（使い切ると次の枠まで動かせない）。
  • Shift＋Enterの改行と半角@でのファイル指定が便利。成果物は内蔵のファイルビューア／ブラウザプレビューで確認する。

Glossary

用語メモ

作業フォルダClaude Codeがファイルを読み書きする対象のフォルダ。「フォルダを開く」から自分のPC内の任意のフォルダを指定する。
許可をバイパスツールの実行を一つずつ人間に確認せず自動で進めるモード。誤ってファイルを消すなどのリスクがあり、初心者はオフのままが推奨。
Sonnet / Opus / HaikuAnthropicのClaude AIモデル。基本はバランス型のSonnet、込み入ったタスクはOpus、軽い処理は高速なHaikuと使い分ける。
思考レベル（低／中）推論モデルが考える深さの設定。速さ重視なら「低」、じっくり考えさせたいなら「中」を選ぶ。
@でのファイル指定プロンプト内で半角の@を付けてファイルを選ぶと、そのファイルを参照対象として指示に含められる便利な書き方。
HTMLWebページなど画面表示のためのマークアップ言語。Claude Codeに作らせ、内蔵のブラウザプレビューで表示を確認できる。
