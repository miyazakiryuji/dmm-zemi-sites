EP01
コースの全体像とゴール

むずかしいコードは書かず、ふだんの日本語で頼みながらアプリやサイトを少しずつ仕上げていく——この入門コースが向かう先と全体像を、まずは最初に整理します。

このコースは、Claude Code をこれから触る人や、以前の学習で行き詰まった経験がある人でも無理なく進められるように組み立てています。いきなり専門的なコードを打ち込むのではなく、Claude Code に向かって「こういうものを作りたい」と日本語で相談し、AI とやり取りしながら手頃なアプリやサイトを一緒に組み上げていく、というやり方です。この第1回では、今後どんな内容を扱い、どこを目的地とするのか、その全体マップを先に頭へ入れておきましょう。

Learn

この回の学習ノート

1このコースが目指していること

このコースの根っこにあるのは、「自分でも作れるかもしれない」という手応えを持ってもらうことです。プログラミングの経験がなくても、Claude Code へ日本語で頼めば、AI と相談しながら思いついたことを少しずつ形にしていけます。最初から完成度の高いものを目指す必要はありません。まず相談して動かし、自分の目で確かめ、気になったところを少し手直しする——この往復のリズムに慣れることが第一のねらいです。

  • Web サイトや Web アプリ、ふだんの作業がラクになる小さな道具を、実際に手を動かしながら作る
  • むずかしいコードを丸暗記するのではなく、まずは口頭の指示で動かしてみる体験から入る
  • 途中で詰まる場面が出ても、ひとつずつ確かめながら進めれば十分、という構えで臨む
最初から何もかも自力で書けるようになる必要はありません。「相談する → 動かす → 確かめる → 少し直す」をぐるぐる回しているうちに、自然と扱える範囲が広がっていきます。

2何を作りながら学ぶのか

このコースは説明をただ聞くだけでは終わりません。回ごとに、実際に一つずつ仕上げていく題材が用意されています。題材はどれも生活や仕事に近い具体的なもので、作ったものが画面で実際に動くと、もの作りの面白さや「これは使えそうだ」という実感がわいてきます。

  • 会社案内のような Web サイトの制作
  • 自分専用の秘書アプリ
  • 日々の作業を支える業務効率化ツール
  • 漫画制作アプリ
説明を聞くだけの座学にとどまらず、自分の手で実際に操作するハンズオン形式も組み込まれています。読み流して終わりにせず、同じ手順をぜひ自分でもなぞってみてください。

3学習の進み方（ロードマップ）

学習は、いきなり高い壁に挑むのではなく、できることを一段ずつ積み上げていく流れになっています。はじめに土台を整え、そこからサイト作り、Web 開発、業務効率化へと、扱う範囲を順々に広げていきます。

  • Claude Code の基本を学ぶ
  • ホームページを作って公開する
  • Web 開発（前半）：秘書アプリ作りを通じて、サービスやアプリ同士をつなぐ仕掛けである API に触れる
  • Web 開発（後半）：入力・出力のできるアプリを作りつつ、API を使ったつなぎ込みをさらに掘り下げる
  • 業務効率化：作業の自動化や、毎日の手間を減らす仕組み作りへと踏み込む
「API」という言葉にとっつきにくさを感じても問題ありません。今は『サービスやアプリ同士を橋渡しする仕掛け』というざっくりしたイメージさえ持っておけば十分で、手を動かしていくうちに少しずつ腑に落ちていきます。

4このコースの2つのゴール

コース全体の到達点は、ざっくり2つにまとめられます。ひとつは「自分で作れるようになること」、もうひとつは「公開や連携までの一連の流れをつかめるようになること」です。この2点を頭の片隅に置いておくと、各回が全体のどのあたりに当たるのかが見通しやすくなります。

  • ゴール1：日本語で Claude Code に指示しながら、Web サイトやアプリを自分の力で作れるようになる
  • ゴール2：作ったものを公開する・API でつなぐ・自動化へ広げる、という流れの全体像をひととおり把握できるようになる
ゴールはあくまで目的地です。途中でつまずいても慌てる必要はなく、ひとつずつ確かめながら進めば大丈夫です。よく分からなかった点は、後の回で実際に手を動かしながら埋めていけます。

Key Points

この回のポイント

  • はじめての人や挫折した経験のある人でも進めやすいよう、コードの暗記からではなく日本語で頼むアプリ作りから入る入門コース
  • 会社案内サイト・秘書アプリ・業務効率化ツール・漫画制作アプリといった、具体的な題材を一つずつ仕上げながら学ぶ
  • 座学だけで終わらせず、自分の手で操作するハンズオン形式も組み込まれている
  • 学習は『基本 → ホームページの作成と公開 → Web 開発 → 業務効率化』の順に一段ずつ積み上がる
  • Web 開発を進める中で、サービスやアプリ同士を橋渡しする API の仕掛けにも触れる
  • 到達点は2つ：日本語の指示でサイトやアプリを自力で作れること／公開・API 連携・自動化までの流れをつかめること
