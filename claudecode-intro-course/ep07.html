EP07
ビジネス利用における注意点

Claude Code を仕事で安心して使うコツは、大きく2つ。秘密の情報を外へ出さないこと、そして判断を AI に丸投げしないことです。日々の作業で見返せる確認の視点までまとめました。

Claude Code はファイルの読み書きやコマンドの実行まで代わりにこなしてくれる頼もしい相棒ですが、できることが多いだけに、扱いを誤ると思わぬトラブルを招きます。この回では、業務で活用するうえで外せない安全の基本を、初学者の方にもわかる形で整理しました。覚えていただきたい考え方はとてもシンプルで、大事な情報を外部へ出さないことと、判断を AI に委ねきらないこと、この2つに尽きます。

Learn

この回の学習ノート

1原則1：情報を外に漏らさない

業務で AI ツールを使ううえで最も警戒すべきなのは、大切な情報がうっかり外部へ出ていってしまうことです。これは Claude Code に限らず生成 AI に広く共通する論点ですが、以下の3つを心がけるだけで、漏洩のリスクはぐっと小さくなります。

  • 認証情報をコードの中に直接書かない：API キーやパスワードといった「鍵」をプログラムへそのまま打ち込むと、そのコードを共有したり公開したりした瞬間に第三者の手に渡ってしまいます。鍵はコードから切り離した場所で保管し、実行時に必要なぶんだけ呼び出す形にしましょう。
  • 機密情報を AI に入力・送信しない：お客様の個人情報や社外秘の資料などを入力欄へ貼り付けると、そこが漏洩の起点になりかねません。送る前に「これは外へ出してよい中身か」と一度立ち止まって確かめる癖をつけてください。
  • 入力を AI の学習に使わせない：インストールの段階などで「学習に協力するか」を切り替えるスイッチ（トグル）が用意されています。ここをオフにしておけば、自分が打ち込んだ内容が学習素材に回らないよう設定できます。
「認証情報」とは、ログインや外部サービスへの接続のときに使う重要な情報をひとまとめにした呼び方です。なかでも API キーは、外のサービスへつなぐための「鍵」だとイメージすると腹落ちしやすいでしょう。
これは Claude Code だけに当てはまる話ではありません。ChatGPT をはじめ、他の生成 AI を使うときも同じように気をつけたいポイントです。どのツールが相手でも「秘密の情報は渡さない」を貫いてください。

2原則2：AI に任せきりにしない（権限モードの考え方）

Claude Code はファイルを書き換えたりコマンドを走らせたりできるので、作業の途中で「この操作を進めてよいですか」と確認してくる場面があります。この振る舞いは「権限モード」、つまりどこまで自動で任せるかを決める設定によって変わります。初期状態では、ツールを最初に使う場面で許可を求めてくるようになっています。

確認の求められ方は、ざっくり次のような段階に分かれます。いま自分がどの程度まで任せているのかを把握して使い分けることが、安全への入り口になります。

  • 標準（確認あり）：ひとつ操作するごとに許可を尋ねてくるモード。最初はここに慣れるところから始めるのがおすすめです。
  • Plan Mode（計画モード）：いきなり手を入れるのではなく、まず Claude に段取りを考えてもらい、「この進め方でよいか」を人が承認してから着手します。読み込みはしても書き換えはしないので、まず様子をうかがいたいときに役立ちます。
  • Auto Mode（自動寄り）：毎回の承認が重なって人がボトルネックになるのを防ぐため、問題なさそうな作業については確認の一部を飛ばして進めます。手数は減る一方、意図しない変更が紛れ込む可能性も上がります。
  • 確認なしの自動実行：承認を介さずに走らせる、最も強力なモード。事故を起こさない備えが要るうえ判断も難しいので、慣れるまでは手を出さないのが賢明です。
「ファイルを編集します」「このコマンドを実行します」といった表示が出たら、それが自分の依頼した内容と食い違っていないかを見てから許可しましょう。はじめは許可できる範囲がせまいので OK のまま進めても大丈夫ですが、少しずつ「いま何を許しているのか」に目を通す習慣をつけると安心です。
操作の権限を広げすぎると、Claude Code が無関係なファイルまでのぞきに行ったり、大切な情報を消してしまったりする恐れがあります。必要以上の権限は渡さないよう注意してください。

3成果物は鵜呑みにせず、人間が最終確認する

AI のアウトプットがつねに正しいとは限りません。生成されたアプリや文章は、必ず人が実際に動かして確かめ、「これは自分で責任を負える中身か」と問い直すことが欠かせません。動作のチェック自体を AI に頼むこともできますが、最後に責任を引き受けるのは人間です。

  • できあがったものを実際に走らせ、思ったとおりに動くかを確かめる
  • 重大なバグが残っていないかを点検する
  • 重要な変更に入る前にはバックアップを取り、万一に備える
バックアップやバージョン管理には、エンジニアの定番ツールである GitHub というクラウドサービスがよく使われます。GitHub は誰でも閲覧できる「公開」状態にもできるため、認証情報入りのコードをそのまま上げてしまうと API キーを悪用される危険があります。いまは「そういうサービスがあるんだな」と頭の片隅に置いておけば十分です。

4使うたびに見返すチェックの観点

ここまでの注意点は「設定」「情報の管理」「成果物」という3つの観点に束ねておくと、折にふれて見返したときに抜け漏れを防げます。Claude Code を使うたびに、ここへ立ち返ってみてください。

  • 設定について：AI 学習に使わせない設定をオンにし忘れていないか／必要以上の操作権限を渡していないか
  • 情報の管理について：パスワードや API キーをコードへ直書きしていないか／GitHub などへ上げる前に環境設定ファイルを除外したか／顧客情報や社外秘を入力していないか
  • 成果物について：できたものを実際に動かして確認したか／重大なバグはないか／重要な変更の前にバックアップを取ったか
やるべきことが多くて気が重く感じるかもしれませんが、ソフトウェア開発には「守るとよい定石（ベストプラクティス）」が積み上がっています。それさえ押さえれば、Claude Code は仕事でも頼れる道具になります。なお、認証情報を渡さない仕組みづくりや誤送信を防ぐ具体策は、これからの回でじっくり扱っていきます。

Key Points

この回のポイント

  • 安全に使う2大原則は、大事な情報を外へ出さないことと、判断を AI に丸投げしないこと
  • API キーやパスワードなどの認証情報はコードへ直書きせず、別管理にして必要なときだけ呼び出す
  • 顧客情報や社外秘などの機密情報は AI に入力・送信しない（生成 AI 全般に共通する注意点）
  • インストール時のトグルで AI 学習をオフにし、必要以上の操作権限は渡さない
  • 権限モードは段階的（標準／Plan Mode／Auto Mode／確認なしの自動実行）。初学者は確認しながら1つずつ進める
  • 成果物は鵜呑みにせず人が動作確認・レビューし、最後の責任は人間が負う
