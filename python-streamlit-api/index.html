Python / Streamlit / API
生成AIアプリ開発の「3つの土台」

生成AIアプリは、この3つの技術の組み合わせでできています。EP17 の内容をベースに、補足を加えて役割と関係をやさしく整理します。EP17＝教材の内容／補足＝発展メモ。

Overview

なぜ この3つ なのかEP17

生成AIアプリ＝AI自体を作るのではなく、すでに提供されている AI（API）を「呼び出して」使うアプリです。それを組み立てるのが次の3つ。

  • 🐍 Python＝アプリを動かす「処理を書く言語」
  • 🎨 Streamlit＝Pythonだけで「画面（UI）」まで作れる道具
  • 🔌 API＝外部のAI（Claude / Gemini / ChatGPT）を「呼び出す窓口」

💡 補足

  • 文法を覚える必要はありません。コードは Claude Code が書きます。大事なのは「どれが何の役割か」を頭に描けること。
  • この回（EP17）は“概要をつかむ”回。実際のインストールは EP18、最初のアプリ作りは EP19 で扱います。
🐍

Python EP17

アプリの「処理」を書くプログラミング言語

  • AI開発で最もよく使われる言語の1つ。
  • Gemini / ChatGPT / Claude などのAIを開発者が使うための Pythonモジュールが必ず配布されるため、AI開発全般で使われています。
  • 特徴：シンプルで読みやすい文法 → プログラミング初心者に人気。
  • 強み：AI／機械学習のライブラリが豊富で、コミュニティも活発。
  • JavaScript などでも生成AIアプリは作れますが、Python が最もメジャーです。

💡 補足

  • 各AIを呼ぶ代表的なライブラリ：openai（ChatGPT）／anthropic（Claude）／google-generativeai（Gemini）。
  • ライブラリの追加は pip install ライブラリ名。Claude Code が必要なものを判断して入れてくれます。
  • 本ゼミでは Python の文法学習は目的ではありません。「何をするものか」が分かれば十分です。
🎨

Streamlit EP17

Pythonだけで「画面（UI）」まで作れるフレームワーク

  • 「Python だけで UI もロジックも書ける AIアプリ開発フレームワーク」。
  • Python のみだとロジックしか書けない → Streamlit で UI も同じファイルに書けます。
  • HTML / CSS の知識がなくても、一定品質のデザインが作れます。
  • UIのボタンを押すたびに、裏で書いた Python のロジックが実行されます。
  • デモ・社内ツール・プロトタイプ開発に最適。
  • 一方、ユーザー認証など複雑な機能は苦手（そこは別の言語／フレームワークで扱います）。
import streamlit as st

text = st.text_input("入力")
if st.button("送信"):
    st.write(f"あなたは {text} と書きました")

💡 補足

  • 起動は streamlit run app.py。実行するとブラウザが自動で開いて画面が表示されます。
  • よく使う部品：st.title（見出し）／st.text_input（入力欄）／st.button（ボタン）／st.write（表示）／st.sidebar（サイドメニュー）。
  • 本格的なWebサービス（ログイン・複雑な画面遷移）に育てたくなったら、React / Next.js などへ移行します。
🔌

AI API EP17

AIの頭脳を「外部から呼び出す」仕組み

  • ChatGPT / Gemini / Claude などのAI機能を、自分のアプリの中で使うために利用します。
  • 無料枠ありの Gemini から、従量課金の OpenAI / Claude まで様々。
  • 生成AIアプリ＝AI自体を作っているのではなく、提供されている API を「呼び出している」だけ。
  • API は AI 以外にも YouTube API・X API などがあり、駆使すれば開発の幅が大きく広がります。

💡 補足

  • APIキー＝そのサービスを使うための「自分専用の鍵」。リクエストに付けて本人確認します（取得は EP21 で扱います）。
  • 従量課金に注意：使った分だけ料金がかかります。無料枠や利用上限の設定を活用しましょう。
  • APIキーは秘密に：他人に見せない・GitHubに上げない（.env などで管理）。
  • 短時間に呼べる回数の上限（レート制限）がある点も覚えておくと安心です。

Architecture

技術スタック全体図（3層構造）EP17

ユーザーの操作から AI の応答までを「層（レイヤー）」で整理した設計図です。

👤 ユーザー↓ 操作
フロントエンド（UI）
Streamlit画面・入力・ボタン
↓ 呼び出し
バックエンド（ロジック層）
Python ＋ Streamlit処理を書く
↓ API 呼び出し
AI API
Gemini / OpenAI / ClaudeAIの頭脳
↑ 応答ユーザーへ表示

💡 補足

  • これを 3層構造（フロントエンド／バックエンド／データ層） と呼びます。AIが発達しても、この設計図を頭に描けないと意図通りのアプリは作れません。
  • Python → JavaScript、Streamlit → React / Next.js などに置き換えても、考え方は同じです。
  • 「データ層（データベース）」は本ゼミの後半（EP31〜32あたり）で登場します。今は2層（UI＋ロジック）＋API、と捉えてOK。
  • この設計図を持っておくと、Claude Code への指示も的確になります（「Streamlitの画面に入力欄を足して、送信したら Claude API を呼んで結果を表示」など）。

Summary

ひとことでまとめ

🐍 Python で処理を書き、🎨 Streamlit で画面を作り、🔌 API で外部のAIを呼ぶ。

この3つを組み合わせると「入力 → AIが処理 → 画面に表示」という生成AIアプリになります。

文法は覚えなくてOK。役割と 3層構造 のイメージさえ持てれば、あとは Claude Code が形にしてくれます。次は EP18（Pythonのインストール） → EP19（最初のアプリ作り） へ。
