「要件を満たす」×「使う子にやさしい」作り方

01

課題の概要

「対象年齢・テーマ・話し方」を選ぶと、その子に合わせて会話してくれる子供向けチャットボットを Dify で作る課題です。次の3つの要件を満たすことがゴールでした。

要件 1

基本設定

チャットボットの名前と説明文を、「何に使うボットか」が一目で分かるように設定する。

要件 2

入力フォーム（4項目）

対象年齢・会話テーマ・話し方のトーン・追加指示の4つを用意。追加指示だけ任意、ほかは必須にする。

要件 3

フォローアップ

回答が表示されたあとに、次の質問の候補ボタンを出して、続けて会話しやすくする。

この課題のねらい： 「すごいボットを作る」ことよりも、まず提示された要件を正確に形にすることが目標でした（教材にも「凝りすぎなくてOK。まず要件を満たそう」とあります）。だからこのレビューも、要件を満たせているかを土台に、その上で光った工夫を紹介します。
02

評価の観点

課題の3要件を、Difyの設定画面のどこに当たるかと合わせて、もう少し具体的に分解します。各作品はこの表を“ものさし”にして見ています。

要件具体的にやることDifyのどこ？
1. 基本設定 名前と説明文で「対象年齢・テーマ・話し方を選べる子供向けの会話ボット」だと一目で分かる。 アプリ名 ／ 説明
2-a 対象年齢 数値入力・必須（例：5、8、10）。 入力フィールド「数値」
2-b 会話テーマ 短文入力・必須（例：「動物の名前」「宇宙の話」）。 入力フィールド「短文」
2-c 話し方のトーン 選択式（5択）・必須。下の5つを用意：
①丁寧でやさしい口調 ②明るく元気な話し方 ③落ち着いた説明スタイル ④お兄さん/お姉さん風 ⑤フレンドリーでカジュアル		 入力フィールド「選択」
2-d 追加指示 段落入力・任意（長さや避けたい言葉、キャラ設定などを自由記入）。 入力フィールド「段落」
3. フォローアップ 回答後に次の質問候補を出す（オン）。 機能 →「次の質問の提案」
用語ミニ解説（初学者向け）：
システムプロンプト：AIへの「役割・ルール」の指示文。ここの作り込みで応答の質が決まります。
入力フィールドの種類：「数値／短文／段落／選択」など。中身に合った種類を選ぶのがコツ。
フォローアップ（次の質問の提案）：回答の後に出る“次はこれ聞く？”ボタン。会話が続けやすくなる。
オープニングメッセージ：最初に表示される一言。ここで子供が話し出しやすくなります。

03

良かった提出（3作品）

3要件をしっかり満たしたうえで、それぞれ別の角度で「使う子供のこと」を考えられていた作品です。真似したいポイントを中心に紹介します。

GOOD 01 要件を完全達成＋「安全」と「年齢チェック」まで配慮

ひとことで：子供が使うことを本気で想定し、要件＋αで“守る仕組み”まで組み込んだ作品。

対象年齢：数値 ✓ 会話テーマ：短文 ✓ トーン：5択 ✓ 追加指示：段落 ✓ フォローアップ：ON ✓ ＋ 個人情報・NGワード対策 ＋ 年齢の範囲チェック ＋ 音声入力 / 専用アイコン 
## 役割 児童心理学と幼児教育にくわしい小学校の先生

## 年齢チェックのルール
{{age}} が「5未満」または「12超」のときは、会話せず固定メッセージで案内：
  「ごめんね！このチャットは 6さい〜12さい のお友達専用なんだ。もういちど年齢をたしかめてね！」

## テーマの絞り込み 「宇宙」など広いテーマは、子供が喜ぶ1つ（例：無重力・宇宙食）に絞る

## 出力 絵文字を散りばめ／3〜4文以内／1〜2文ごとに改行 …

さらに：不適切ワード対策（コンテンツモデレーション）も設定し、子供が名前・住所・電話番号などを書いたり、乱暴な言葉を使ったときは、やさしい定型文でそっと止めるようにしていました。

✓ ここが良い
  • 要件をすべて達成（しかも追加指示を「段落」入力に）：4項目の入力タイプが要件どおりで、3作品の中でも特に正確。
  • “使う子供を守る”発想：個人情報・乱暴な言葉のフィルタ、対象年齢の範囲チェックまで設計。子供向けだからこその配慮が光る。
  • 役割（ペルソナ）が具体的：「児童心理の専門家＝小学校の先生」と決めることで、言葉づかいが一貫する。
GOOD 02 要件達成＋「最初のあいさつ」で子供が話し出しやすい

ひとことで：システムプロンプトの型がきれいで、オープニングメッセージの工夫が秀逸な作品。

対象年齢：数値 ✓ 会話テーマ：短文 ✓ トーン：5択 ✓ 追加指示：任意 ✓ フォローアップ：ON ✓ ＋ ひらがな＋絵文字のあいさつ文 
▼ オープニングメッセージ（最初に表示される一言）
こんにちは！😊
ぼく（わたし）は、みんなのおはなしを聞いたり、しつもんにこたえたりするおともだちだよ！
たとえば、
🌟「きょうりゅうのことがしりたい！」
🌟「さんすうをおしえて！」
🌟「たのしいクイズをだして！」
みたいに、おはなししてくれるとうれしいな。
✓ ここが良い
  • 最初の一言で会話のハードルを下げている：全部ひらがな＋絵文字＋具体例で、子供が「何を聞けばいいか」すぐ分かる。空欄にしがちな所を丁寧に作り込んだ好例。
  • システムプロンプトの型がお手本：「役割／目的／入力情報／出力要件／注意点」と整理され、4つの変数（年齢・テーマ・トーン・追加指示）をきちんと使っている。
  • 要件をすべて達成：入力4項目・必須/任意・トーン5択・フォローアップONまで過不足なし。
GOOD 03 要件達成＋「AIの出力ルール」を具体的に決めて応答が安定

ひとことで：応答の長さ・言葉づかい・禁止事項まで指示し、答えがブレない作品。

対象年齢：数値 ✓ 会話テーマ：短文 ✓ トーン：5択 ✓ 追加指示：任意 ✓ フォローアップ：ON ✓ ＋ 説明文も分かりやすい 
## 出力要件
- 子供が自然に理解できる言葉づかいで書く
- 応答は1ターン分（数文以内）。長すぎず短すぎず
- 難しい言葉は、かんたんな説明を添えるか言い換える
- 見出しや箇条書き（Markdown）は使わず、会話文だけで返す
## 注意点
- 年齢に合わない言葉や、むずかしい抽象表現は避ける
- 否定・命令・きつい表現は避け、やさしく肯定的に
✓ ここが良い
  • 「どう答えるか」のルールが具体的：長さ・言い換え・会話文のみ…とAIの出力を細かく方向づけているので、毎回の応答が安定する。
  • “やってはいけないこと”も明記：否定・命令・きつい表現を避ける、と書くことで子供向けにふさわしいトーンを保てる。
  • 要件をすべて達成：入力の種類・必須/任意・5択・フォローアップまできちんと設定。
良かった3作品の共通点： どれもまず3要件を確実に満たし、そのうえで「安全」「最初のあいさつ」「出力ルール」と、使う子供の体験を一歩ずつ良くする工夫を足していました。土台（要件）＋ひと工夫、が伸びる作品の形です。
04

あと一歩の提出（3作品）

どの作品も良いところがしっかりあります。要件のうち1〜2か所を直すだけで「良かった作品」に届くものばかりなので、まず良い点 → 次に直すと伸びる点の順に見ていきましょう。

あと一歩 01 入力の作りは正確。あとは「説明文・あいさつ・候補の指示」

傾向：入力フォームとシステムプロンプトはきちんとできている。仕上げの“見せ方”が少し惜しい。

対象年齢：数値 ✓ 会話テーマ：短文 ✓ トーン：5択 ✓ 追加指示：任意 ✓ フォローアップ：ON ✓ 説明文：空 ✗ あいさつ文：なし △
◎ まず、ここが良い
  • 入力フォームが要件どおり正確：対象年齢を「数値」、トーンを「5択」で作れており、必須/任意の区別もOK。基礎がしっかりしている。
  • システムプロンプトの型が整っている：役割・目的・入力情報・出力要件…と構造化され、変数も正しく使えている。
△ あと一歩（直すと伸びる）
  • 説明文が空：要件1「一目で用途が分かる」が片手落ち。一覧で何のボットか分からなくなる。
  • 最初のあいさつ（オープニング）が無い：子供が「何を話せばいい？」と迷いやすい。
  • フォローアップの指示文が“設定の説明文そのまま”：「次の入力候補を提案するボタンを表示」と書いてある＝これはAIへの指示になっていない。
Before

説明文＝空／あいさつ＝無し／候補の指示＝「ボタンを表示」とだけ。

After（こう直す）

説明文に「対象年齢・テーマ・話し方を選べる子供向けの会話ボット」を一文。あいさつにひらがなの一言＋例。候補の指示は「15文字以内・ひらがな多め・子供が選びやすい質問を3つ」に。

あと一歩 02 4項目そろってる。「年齢＝数値」にできれば完成度アップ

傾向：要件の形はほぼ満たしている。入力の“種類”の選び方がもう一歩。

対象年齢：短文 △ 会話テーマ：短文 ✓ トーン：5択 ✓ 追加指示：任意 ✓ フォローアップ：ON ✓ 名前・説明文：あり ✓
◎ まず、ここが良い
  • 必要な4項目とトーン5択がそろっている：必須/任意も要件どおりで、フォローアップもON。要件の全体像をきちんと押さえている。
  • 名前・説明文が設定済み：何のボットかが分かる。会話の流れ（あいさつ→テーマ）もプロンプトで設計できている。
△ あと一歩（直すと伸びる）
  • 「対象年齢」が短文入力：要件は「数値入力」。数値にすると「5」「8」のように入り、年齢で出し分ける処理も安定する。
  • 追加指示が1行（短文）：本来は「段落」入力。長めの指示（避けたい言葉やキャラ設定）が書けるようになる。
  • 追加指示の“説明文”をプロンプトに貼り付け：変数のすぐ後に説明文が続いて役割が混ざりぎみ。説明は入力欄のラベル側へ。
Before

対象年齢＝短文／追加指示＝短文（1行）。

After（こう直す）

対象年齢の種類を「数値」に変更。追加指示を「段落」に変更。説明は欄のラベル/プレースホルダに移すと、プロンプト本文がすっきりする。

あと一歩 03 子供目線のUXが一番豊か。あとは「名前・型・プロンプト整理」

傾向：「子供が使う」イメージが3作品で一番できている。仕上げの整理で大きく伸びる。

ひらがなラベル 最初の話題ボタン 音声入力 ON トーン：5択 ✓ フォローアップ：ON ✓ 名前：あいまい △ ねんれい：短文 △
◎ まず、ここが良い（ここは積極的に真似したい）

△ あと一歩（直すと伸びる）
  • アプリ名が「こども向け」だけ：用途が一目で伝わりにくい（要件1）。会話ボットだと分かる名前に。
  • 「ねんれい」が短文入力：要件は数値入力。
  • システムプロンプトに“別のボットの名残”と重複：「職業・業界・役職になりきって…」など別用途の文や、同じ見出し（追加指示／出力要件）が2回出てくる。コピペの整理を。

名前を「子供向け会話チャットボット」等に。年齢を「数値」へ。プロンプトは1本の筋に整理（別用途の文・重複見出しを削除）。ラベルも日本語に。中身の発想は良いので、整えるだけで一気に良作。

05

全体の総括

今回の作品を通して見えた「伸びている作品の共通点」と、「あと一歩の作品に多いつまずき」をまとめます。どれも“次に直せばすぐ良くなる”ものばかりです。

伸びている作品の共通点

まず3要件を“正確に”満たす

入力の種類（数値・短文・段落・選択）を中身に合わせて選び、必須/任意・トーン5択・フォローアップまで漏れなく設定している。

使う子供の目線で整える

やさしい言葉のあいさつ文、安全への配慮、読みやすい出力ルールなど、「使う子のこと」をひと手間考えている。

システムプロンプトが整理されている

「役割／目的／入力情報／出力要件／注意点」の型で、変数を全部使い切る。余計な文が混ざっていない。

あと一歩の作品に多い点

名前・説明文が空／あいまい

一覧で何のボットか分からない。一文の説明を足すだけで要件1が埋まる。

入力の「種類」が惜しい

対象年齢が文字入力のまま／追加指示が1行のまま。数値・段落に変えるだけで要件どおりに。

プロンプトの“コピペ残り”と指示の流用

別用途の文や見出しの重複、フォローアップ欄に設定の説明文をそのまま。整理すると一気に伝わる。

このレビューの一番の学び： 良い作品ほど、特別なことをしているわけではありません。①3要件を正確に満たす → ②入力の“種類”を中身に合わせる → ③使う子供の目線でひと工夫、という順番を踏んでいるだけです。あと一歩の作品も、良い発想はすでに持っています。要件の抜けを1つずつ埋めれば、しっかり良作になります。

提出前セルフチェックリスト

次にチャットボットを作るとき、提出前にこの順番で見直すと“あと一歩”を埋められます。

まず要件（ここが土台）

  • ✓名前・説明文で「何のボットか」が一目で分かるか（説明文が空になっていないか）
  • ✓対象年齢＝数値／会話テーマ＝短文／話し方トーン＝選択(5つ)／追加指示＝段落で作れているか
