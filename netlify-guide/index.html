Chapter 01 3 ステップ

アカウントを作成しよう

Netlify公式サイトにアクセスし、サインアップから氏名登録までを行います。

01

Netlify公式サイトにアクセス

ブラウザで https://www.netlify.com を開きます。 右上の 「Sign up」 ボタン、もしくは中央の 「Start building」 をクリックして登録に進みましょう。

💡
ヒント

どちらのボタンを押しても、同じサインアップ画面に進めます。

02

サインアップ方法を選ぶ

「The launch center for first-time builders」 の画面が表示されます。 利用するアカウントを選んで進みましょう。

  • ✓ Googleでサインアップ(おすすめ・最も手軽)
  • ✓ GitHubでサインアップ(開発者向け)
  • ✓ GitLab / Bitbucket(他Gitホスティング利用者)
  • ✓ Emailでサインアップ(SNS連携したくない場合)
03

名前を入力する

「Nice to meet you! Let's get acquainted.」 の画面で、First name(名)と Last name(姓)を入力します。 ここから複数の質問にステップ形式で答えていきます。

📌
入力例

First name: Ryuji / Last name: Miyazaki のようにローマ字で入力するとシンプルです。

Chapter 02 6 ステップ

オンボーディング設定

Netlifyからの簡単な質問に答えていきます。回答内容に応じて、最適な案内が用意されます。

04

利用目的を選ぶ

「How are you planning to use Netlify?」(どのように利用しますか?) の質問に答えます。

🏢
Client
クライアント案件
🎓
School
学校・授業
👤
Personal
個人利用
💼
Work
仕事
05

Web開発の経験レベルを選ぶ

「What's your experience building on the web?」(Web開発の経験はどのくらいですか?) に回答します。 4段階から正直に選びましょう。

Beginner — 開発自体がはじめて
Basic — シンプルなプロジェクト経験あり
Intermediate — 定期的にビルド・公開している
Advanced — 複雑なプロジェクトを設計できる
06

最初に作るプロジェクトの種類を選ぶ

「What kind of project do you want to build first?」(最初に何を作りますか?) を選びます。 ポートフォリオなら Personal project がおすすめです。

Blog Web app Prototype/MVP Personal project Documentation Games School assignment Learning/experimenting eCommerce
07

自分の役割を選ぶ

「What best describes your role?」(あなたの役割は?) で、最も近いものを選択します。 該当しない場合は Other でOKです。

08

Netlifyを知ったきっかけを選ぶ

「How did you hear about us?」(どこでNetlifyを知りましたか?) を選びます。

🔍
SearchGoogle, DuckDuckGo
🤖
AI or code toolClaude, Bolt
👥
Someone else同僚・友人
💬
SocialX, Reddit
09

チーム名を入力する

「What is the name of your team?」 で、チーム名を入力します。 個人利用の場合は 自分の名前やプロジェクト名 を入れるのが一般的です。

⚠️
あとから変更可能ですが…

チーム名はURLの一部などに使われることがあるため、シンプルでわかりやすい名前にしておくのがおすすめです。

Chapter 03 5 ステップ

最初のプロジェクトをデプロイする

いよいよ実際のサイト公開へ。ファイルをドラッグ&ドロップするだけでデプロイできます。

10

「Continue to deploy」で次に進む

質問への回答が完了したら、画面下部に 「Continue to deploy」 ボタンが現れます。 これをクリックすると、いよいよデプロイ画面へ進みます。

11

デプロイ方法を選ぶ

「Deploy your first project.」 の画面で、デプロイ方法を3パターンから選択できます。

A
AIエージェントで作る

Claude Agent等を使い、プロンプトから自動でサイト生成。400 Extra Credits がもらえる。

B
Gitリポジトリから取り込む

GitHub / GitLab / Bitbucket / Azure DevOps から既存リポジトリを連携。

C
ファイルを直接アップロード

HTML/CSSの完成ファイルをドラッグ&ドロップ。今回はこの方法を使います。

12

ファイルをアップロードする

「Upload your project files」 エリアに、プロジェクトフォルダ・ZIP・単一HTMLファイルをドラッグ&ドロップします。 browse files to upload から選択することも可能です。

📦
用意するファイル

index.html をエントリーポイントとして含むフォルダ、もしくはZIPがあればOKです。

13

index.html へのリネーム確認

アップロードしたファイル名が index.html でない場合、 「Rename to index.html?」 のダイアログが表示されます。 「Rename and deploy」 を選ぶと、自動でリネームしてデプロイされます。

⚠️
なぜ重要?

Webサイトのトップページは index.html である必要があります。これがないと「Page Not Found」エラーになります。

14

デプロイ完了 🎉

アップロードが完了すると、自動的に プロジェクトダッシュボード に遷移します。 ランダムなプロジェクト名(例: glowing-sawine-2df6e3)で公開されます。

🚀
あなたのサイトが世界に公開されました!

https://*****.netlify.app でアクセス可能です。

Chapter 04 3 ステップ

公開後のカスタマイズ

プロジェクト名やドメインを整えて、シェアしやすいサイトに仕上げます。

15

プロジェクト名を変更する

ダッシュボードの 「Quick setup」 から 「Update the project name」 を開きます。 プロジェクト名はそのままURLになるので、覚えやすい名前にしましょう。

https:// miyazaki-portfolio .netlify.app
📝
命名ルール

英数字と単一のハイフン(-)のみ使用可。最大37文字です。

16

カスタムドメイン・パスワード保護を設定

次の画面で、より上級のカスタマイズを選べます。必要なければスキップでOKです。

🌐

Set custom domain

独自ドメイン(例: example.com)を接続できます。 所有しているドメインを追加、または新規購入も可能。

🔒

Set password protection

サイトをパスワードで保護できます。クライアント確認用や、初期段階の制作物の限定共有に便利。

17

プロジェクト一覧で確認 🎊

すべての設定が完了すると、プロジェクト一覧に サムネイル付きのカード が表示されます。 緑のドットが Published 状態を示しています。

🎉
これで Netlify 利用開始は完了です!

あとは作りたいサイトをどんどん公開していくだけ。Have fun shipping!

🎉✨🚀💫🎊

お疲れさまでした!
