Claude Code セッションの中断・再開方法

作業を一度やめても、会話の続きからまた始められます。「やめ方（中断）」と、2 つの「再開のしかた」（--continue と --resume）、その使い分けまでをコマンド例つきでまとめました。

Introduction

はじめに

Claude Code とは？ ターミナル（CLI）から使える AI コーディングツールです。コードを書く・直す・調べるといった作業を、対話しながら一緒に進められます。

作業はいつも一度で終わるとは限りません。長い作業を区切って進めたり、続きを別の日にやったり、途中で席を立ったり。そんなときに「中断・再開」を覚えておくと、それまでの会話履歴や文脈をそのまま引き継いで、前回の続きからスムーズに作業を再開できます。

  • 毎回ゼロから説明し直さなくていい — 前回の文脈をそのまま引き継げる
  • 長い作業を安心して区切れる — 終了しても会話は自動保存される
  • 「あの会話の続き」に戻れる — 過去のセッションを選んで再開できる

Stop & Exit

1. セッションの中断（終了）方法

まずは「やめ方」から。Claude Code を立ち上げている状態（対話プロンプト）から抜けて、いつものシェルに戻る方法です。

会話履歴は自動で保存されます。 どの方法で終了しても、ここまでのやりとりは自動保存され、あとから再開できます。手動でのセーブ操作は不要です。

Ctrl+C×1 中断のみ（終了しない）。実行中の AI の返答やツールを止めて、空のプロンプトに戻ります。セッションは開いたまま。
Ctrl+C×2 終了する。素早く 2 回押すとシェルに戻ります。1 回押すと「もう一度押すと終了します」とヒントが出ます。
Ctrl+D 終了する。入力が空の行で押すと終了。こちらは 1 回でOKです。
 /exit 終了する。/exit（または /quit）と入力して Enter。一番わかりやすい終了方法です。

「1 回押したのに終わらない！」 は正常です。1 回の Ctrl+C は “処理の中断” で、うっかり作業中のセッションを終了しないための安全装置になっています。終了したいときは 2 回押すか /exit を使いましょう。

Resume #1

2. ① 直前の続きをすぐ再開：claude --continue（-c）

「さっきの続きをやりたい」だけなら、これが最速です。今いるフォルダで一番新しい会話を、選択画面なし・一発で再開します。

  • claude -c は、今いるディレクトリで直近に使っていた会話をそのまま開きます（選ぶ画面は出ません）
  • 「とりあえず前回の続き」をワンコマンドで再開したいとき向け
  • 対話画面（REPL）に入って再開します。1 問だけ実行して終わらせたいときは -p を付けます

今いるフォルダの一番新しい会話を、選択画面なしで開きます。

--continue の短縮形。動作はまったく同じです。普段はこちらで十分。

-p（--print）を付けると、直近の会話を再開 → 1 つのプロンプトを実行 → 結果を表示して終了します。対話画面には入りません。

注意：見ているのは “今いるフォルダ” の会話だけ。 別のフォルダに cd していると前回の会話は出てきません。また同じフォルダで複数のセッションを動かしていた場合は、いちばん新しい 1 件が開くので、狙って選びたいときは次の --resume を使いましょう。

Resume #2

3. ② 過去の会話を選んで再開：claude --resume（-r）

「いくつかある会話から選びたい」「あの特定の会話に戻りたい」ならこちら。一覧（ピッカー）を出して選べるほか、セッションIDを直接指定して一発で開くこともできます。

  • claude -r で過去の会話一覧が出ます。矢印キーで選んで再開（新しい順に並びます）
  • セッションIDがわかっていれば claude --resume <session-id> で、その会話を直接開けます（一覧は出ません）
  • 検索語を付けると一覧を絞り込めます（指定した文字列は “絞り込みキーワード” として扱われます）
  • 一覧（ピッカー）は対話ターミナルでだけ使えます。-p と組み合わせるときは、一覧の代わりに ID か検索語を渡します

過去の会話の一覧が出るので、矢印キーで選びます。「どれだっけ？」というときの基本形。

--resume の短縮形。同じ一覧が出ます。

特定の会話をピンポイントで開きます。ID は ~/.claude/projects/<フォルダ>/ にある .jsonl ファイル名（UUID）です。

指定した文字列で絞り込んだ状態で一覧を開きます。会話が多いときに探しやすくなります。

存在しない ID を渡すと「見つからない」エラーになります（打ち間違いや、別フォルダの ID など）。新規セッションが始まるわけではないので、ID はコピー＆ペーストで正確に渡しましょう。

Which to use

4. 使い分けのポイント

迷ったらシンプルに。「さっきの続き」なら -c、「どれにするか選びたい／特定の会話に戻りたい」なら -r。下の表で違いをまとめました。

  • とにかく直前の続きをすぐ始めたい → claude -c（選ばない・速い）
  • 複数の会話から選びたい、または昔の特定の会話に戻りたい → claude -r（選べる・ID 指定も可）
  • どちらも “今いるフォルダ” の会話だけが対象。別プロジェクトを続けたいときは、そのフォルダに移動してから実行します
観点 --continue（-c） --resume（-r）
起動の速さ・手数最速。コマンド 1 つで即再開（選ぶ操作なし）一覧から 1 つ選ぶワンステップが入る（ID を渡せば即再開も可）
セッションの選択選べない。直近の 1 件が自動で開く選べる。一覧から矢印キーで選択、または検索語で絞り込み
どんな時に使う「さっきの続きをそのままやりたい」とき「複数の会話から選びたい／特定の会話に戻りたい」とき
コマンドclaude --continue（短縮：claude -c）claude --resume（短縮：claude -r）
直前以外のセッション開けない（常に直近の 1 件）開ける（一覧 or claude --resume <session-id>）
よくある例claude -cclaude --resume 017c2d01-…
もっと便利に使う（応用）
  • --fork-session：再開時に付けると、元の履歴を汚さずに新しいセッションに枝分かれ。claude -c --fork-session や claude --resume <id> --fork-session のように使います。「試しに別方向で進めたい」ときに。
  • --session-id <uuid>：新規セッションに自分で決めた IDを割り当てます。自動化・スクリプトで、あとから --resume <同じID> で確実に続けたいときに便利。
  • --no-session-persistence：その実行を保存しない（-p 専用）。使い捨ての質問向け。保存されないので後から再開はできません。

Where it's saved

5. 会話履歴はどこに保存される？

中断しても再開できるのは、会話が自動でローカルに保存されているからです。仕組みをざっくり知っておくと、トラブル時に落ち着いて対処できます。

会話は 1 行 1 イベントの JSON Lines 形式で、進行に合わせて自動追記されます。

  • 自動保存：保存は自動・常時オン。手動セーブは不要です
  • フォルダ単位で管理：履歴は「作業していた絶対パス」ごとに分かれて保存されます（パスの / を - に置き換えたフォルダ名）
  • ファイル名 = セッションID：.jsonl のファイル名（UUID）が、そのまま claude --resume <session-id> に渡す ID です

「前の会話が見つからない」一番多い原因は “フォルダ違い”。 履歴はディレクトリ単位なので、別の場所に cd していると前の会話は出てきません。まず元の作業フォルダに戻ってから claude -c / claude -r を実行してください。

FAQ

よくある質問

Q.間違ってセッションを終了してしまいました。やり直しできますか？

大丈夫です。会話の履歴は終了時に自動保存されるので、そのフォルダで claude -c を実行すれば直前の続きから再開できます。手動で保存する操作は不要です。

Q.会話の履歴はどこに保存されていますか？

ホーム配下の ~/.claude/projects/<プロジェクトごとのフォルダ>/<session-id>.jsonl に自動保存されます。フォルダ名は「作業していた絶対パスの / を - に置き換えたもの」で、ファイル名（UUID）がそのまま claude --resume <session-id> に渡す ID です。

Q.別フォルダで始めたセッションの続きはできますか？

履歴はフォルダ（ディレクトリ）単位で管理されています。別フォルダに cd した状態だと前の会話は出てきません。「前回の会話が見つからない」ときは、まず元の作業フォルダに移動してから claude -c / claude -r を実行してください。これが “見つからない” の一番多い原因です。

Q./clear した後に再開すると、会話は消えていますか？

消えていません。/clear はその場の文脈（モデルが見ている直近のやりとり）を空にするだけで、ディスク上の履歴ファイルは削除しません。そのため -c や -r で再開すると、/clear する前のやりとりも含めた全履歴が戻ってきます。「クリアしたのに前の話が残ってる」と驚くポイントなので覚えておきましょう。

Q.保存したくない使い捨ての質問もできますか？

できます。claude -p --no-session-persistence "使い捨ての質問" のように、-p（--print）と一緒に --no-session-persistence を付けると、その実行は履歴ファイルに書き込まれません。ただし保存されないので、あとから --continue / --resume で再開することはできません。
