付録：ウェブの先へ ― さらに広がる Claude Code の世界

ここまでのゼミでは、おもにウェブサイトやウェブアプリを作ってきました。この付録では「ウェブ以外」の開発に一歩踏み出す例として、スマホアプリ作りを体験します。あわせて、ゼミを終えたあとに何をどう学んでいくか、これからの道筋を整理します。手を動かす内容と、これからの方針を、どちらもやさしく押さえていきましょう。

1

ウェブ以外の Claude Code 開発（スマホアプリを体験する）

Claude Code が作れるのはウェブサイトだけではありません。ここでは「スマホアプリ」を題材に、ウェブ以外の開発がどんな雰囲気かを体験します。むずかしい専用環境（Flutter や Apple の開発ツール）は使わず、Expo という手軽なしくみと、スマホに入れた Expo Go アプリでその場で動きを確かめます。

  • Claude Code はウェブ以外も作れる。今回の題材は「英単語クイズアプリ」のようなスマホアプリで、ウェブで学んだ感覚をそのままモバイルに広げられる。
  • 今回は Expo（エクスポ）というしくみを使う。Expo は iPhone と Android の両方に対応した（クロスプラットフォームな）アプリを、手軽に作れるのが特長。
  • あえて使わないものもある。Flutter や Apple の開発ツール（Swift・Xcode など）は今回は使わない。これらは高機能だが準備が重く、Mac が必須になる場面もあるため、入門ではより手軽な Expo を選ぶ。
  • 動作確認はスマホの Expo Go アプリで行う。パソコンで作ったアプリを、画面に出る QR コードで読み込むと、自分のスマホ実機ですぐに動かして確かめられる。
  • 作るときは Claude Code に「やりたいこと」を言葉で伝えるだけ。たとえば『Expo で英単語クイズアプリを作りたい。認証もデータベースも不要で、データは端末内に保存、Expo Go でテストできる構成で』のように、欲しい形を素直に頼めばよい。

手順

  1. Cursor で新しいフォルダ（例：English-Quiz-App）を開き、ターミナルで claude と入力して Claude Code を起動する。
  2. Claude Code に「Expo を使ってスマホアプリを作りたい」と伝える。認証やデータベースは不要、データは端末内に保存、Expo Go でテストできる構成、と条件も一緒に書くとシンプルに仕上がる。
  3. Claude Code が実装を進めるあいだ、確認を求められたら内容を見て進めてよければ Yes（許可）を返す。これを数回くり返すと、ひと通り完成する。
  4. スマホ側に App Store / Google Play から「Expo Go」アプリを先にインストールしておく（ここを忘れると QR を読んでも何も起こらない）。
  5. パソコンの新しいターミナルで npx expo start を実行すると QR コードが表示される。iPhone は標準カメラ、Android は Expo Go の「Scan QR code」で読み取ると、自分のスマホでアプリが立ち上がる。

💡 補足・コツ

  • エラーが出たら、エラー文（スマホの画面やターミナルのログ）をそのままコピーして Claude Code に貼り付け、修正をお願いするのが基本の流れ。慌てず1つずつ直していけばよい。
  • QR でうまく繋がらないときは、パソコンとスマホが同じ Wi-Fi にいるかを確認。社内やカフェの Wi-Fi で繋がらない場合は、スマホのテザリングに切り替えると解決することが多い。
  • うまく実機で動かないときの代わりに、パソコン上の Android エミュレーター（仮想スマホ）で確認する方法もある。実機が手元になくても試せる。
  • 今回のアプリはデータを端末内（Async Storage と呼ばれるスマホ内の保存場所）に持つので、外部のデータベースは不要。まずは「自分のスマホで動く」体験を目的にしよう。
  • App Store や Google Play への正式公開は、別途 Apple や Google への開発者登録（有料）が必要。まずは Expo Go で身近な人に試してもらうだけでも、十分に手ごたえを感じられる。
2

今後の学習に向けて（これからの道筋）

ゼミをひと通り終えたあなたは、もう Claude Code を「自分のやりたいことに使える」段階に立っています。ここでは、学びを止めずに伸ばしていくための道筋を、無理のない順番で示します。背伸びしすぎず、まずは身近なことから一歩ずつで大丈夫です。

  • まずは「作りたいものリスト」に戻る。ゼミの中で出した『やりたいこと』『効率化できそうな業務』のアイデアを開き直し、未着手のものから1つ選んで作ってみるのが、いちばん続けやすい次の一歩。
  • 学んだことは、自分の仕事や生活に組み込むと定着する。ウェブサイト、ちょっとした業務ツール、スマホアプリなど、身近な課題を Claude Code で1つ解決してみよう。
  • 余裕が出てきたら、発展的なテーマに少しずつ触れる。たとえばサブエージェントの活用、ハーネス（指示や道具立て）の工夫、コンテキスト（文脈・会話の窓）の管理など。ただし最初は「Claude Code にやりたいことを伝える」だけでも十分。
  • 公開・運用するなら安全面を意識する。とくに API キーのような秘密の情報を外部に漏らさないこと。.gitignore で除外し、もし誤って公開してしまったら、まずキーを無効化（再発行）するのが最優先。
  • わからないことは Claude Code 自身に聞ける。『サブエージェントの使い方を教えて』『セキュリティで気をつけることは？』のように質問すれば、学びながら進められる。公式ドキュメントの確認とあわせて活用しよう。

手順

  1. ゼミで書きためた「やりたいこと」「効率化アイデア」を見返し、いま一番作りたい・役立ちそうなものを1つだけ選ぶ。
  2. 選んだものを Claude Code で実際に作ってみる。小さく始めて、動いたら少しずつ機能を足していく。
  3. 作ったものを公開・共有する前に、Claude Code にセキュリティチェックを依頼し、API キーなどの秘密情報が含まれていないか確認する。
  4. 慣れてきたら、サブエージェントやコンテキスト管理などの発展トピックに、興味のあるものから1つずつ触れていく。
  5. 学んだことを定期的に「自分の業務・生活」に当てはめて使い続ける。使う中で出た疑問は Claude Code や公式情報で解消していく。

💡 補足・コツ

  • 本格的な公開・商用利用や個人情報を扱う場合は、自動チェックだけに頼らず、必要に応じて専門家のレビューも検討する。セキュリティは「ここで終わり」がない領域なので、公開後も定期チェックを習慣にしよう。
  • 作るより「知ってもらう」ほうが大変なこともある。アプリやサイトを公開したら、誰にどう届けるか（紹介・告知の工夫）まで考えると、せっかくの成果が活きる。
  • 最新情報は公式ドキュメントや Anthropic の公式発信を時々のぞくと、新機能やコツに気づける。一度に追い切ろうとせず、気になったときに見るくらいで十分。
  • 学びの区切りに「修了の達成感」を持つのも大切。やり切った課題やリリースを1つの節目として、次のテーマへ気持ちよく進もう。
