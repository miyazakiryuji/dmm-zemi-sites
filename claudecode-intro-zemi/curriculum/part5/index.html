業務効率化 ― Claude Code を「仕事の相棒」にする

Claude Code は開発専用の道具ではなく、毎日のくり返し作業や事務仕事を任せられる「仕事の相棒」にもなります。このパートでは、ファイル整理・資料修正・コンテンツ生成という身近な3つの自動化を、初学者でも手を動かせる形で体験します。

1

Claude Code による業務効率化の事例

Claude Code はプログラミング開発のためだけの道具ではありません。毎日のくり返し作業や事務仕事を「お願いするだけ」で自動化できる、頼れる仕事のパートナーにもなります。まずはどんなことができるのか、イメージをふくらませましょう。

  • Claude Code はパソコンの中のファイル操作・文章作成・データ整理まで任せられる。開発以外の「業務効率化・自動化」にこそ大きな効果がある
  • 代表的な活用例：毎朝のリサーチ（最新ニュースや調べもの）、メールの返信下書き作成、予定表への入力、報告書・議事録づくり、表計算データの変換など
  • 一度きりの作業ならその場でお願いするだけでよいが、毎回くり返す作業はスキル（Skills）として登録しておくと、次からは何度でも同じ品質で再現できる
  • うまくいくコツは「すごいもの」を目指さず、自分が一番時間を取られている小さな作業を1つ選ぶこと。請求書づくり・議事録・メール返信などが定番
  • 機密情報（顧客名簿・社員情報など）を含む業務は、架空のデータに置きかえてから試すか、AIに渡さない判断も大切。安全第一で進める

💡 補足・コツ

  • どんな作業を任せられるか思いつかないときは、直近2〜3日の自分の行動を30分単位でメモに書き出し、それを Claude Code に渡して「ここから自動化できそうな作業を提案して」と聞くのが近道です。
  • 「2回以上くり返している作業」は自動化の有力候補。1つ作って効果を実感してから、次の作業へ少しずつ広げていきましょう。
2

ファイル整理を自動化してみよう

バラバラに散らかったファイルを、名前を変えるだけでなく「中身まで読んで」フォルダ分け・整理してもらいましょう。さらに「どんな基準で整理したか」「各ファイルに何が書かれていたか」をまとめた作業報告書まで出してもらえます。請求書や領収書の仕分けを例に体験します。

  • ふつうのファイル整理ツールは「ファイル名」しか見ないが、Claude Code は中身（PDFや画像の文字）まで読み取って判断できるので、人がやるような整理が可能
  • たとえば請求書・領収書を取引先ごと・月ごとのフォルダに自動で仕分けし、わかりやすいファイル名（例：2026-04_〇〇商事_請求書.pdf）に付け替えられる
  • 整理して終わりではなく、作業報告書として「どんな観点で分けたか」「各ファイルの取引先名・金額・科目」を一覧にまとめて出力できるのが大きな価値
  • 実行前に必ず整理対象のファイルをコピーしてバックアップを取り、まずは少数のファイルで試してから本番に広げると安心
  • 確定申告の領収書まとめ、写真フォルダの仕分け、ダウンロードフォルダの片づけなど、応用範囲は広い

手順

  1. 整理したいファイル（請求書・領収書など）を1つのフォルダにまとめ、必ずコピーしてバックアップを取っておく。
  2. そのフォルダで Claude Code を起動し、何をしてほしいかを言葉で伝える。例：「このフォルダの請求書・領収書を中身まで見て、取引先ごと・月ごとのフォルダに仕分けして、わかりやすい名前に変更してください」。
  3. あわせて報告書も依頼する。例：「どんな観点で整理したか、各ファイルの取引先名・金額・科目を一覧にした作業報告書も出力してください」。
  4. Claude Code が提案する整理プランを確認し、問題なければ実行してもらう。意図と違えば「月別ではなく取引先別を優先して」など言葉で修正する。
  5. できあがったフォルダ構成と報告書を見て確認する。よく使う整理なら、最後に「この手順をスキルにして次から再利用できるようにして」と頼んでおく。

💡 補足・コツ

  • 最初は5〜10ファイルなど少量で試し、整理の基準が自分の思った通りかを確かめてから全件に広げると失敗しにくいです。
  • 「フォルダ分けの基準（取引先別／月別／勘定科目別）」を最初にはっきり伝えると、思い通りの結果になりやすくなります。
3

資料修正を自動化してみよう

すでにある資料を、Claude Code に言葉で指示するだけで修正・整形してもらいましょう。表記ゆれの統一や体裁そろえなど、地味だけれど時間のかかる「直し作業」こそAIの得意分野です。

  • 既存のテキスト・原稿・一覧表などを開いた状態で、「ここをこう直して」と日本語で伝えるだけで一括修正できる
  • 得意なのは、表記ゆれの統一（例：「Cloud Code」を「Claude Code」に直す）、見出しや箇条書きの体裁そろえ、誤字脱字チェック、文体の統一など
  • 「すべてのファイルにまとめて適用」もできるので、何十か所もある同じ直しを一度のお願いで終わらせられる
  • 修正の前に元ファイルのコピー（バックアップ）を取っておけば、気に入らなければいつでも元に戻せて安心
  • 修正後は必ず変更箇所を自分の目で確認する。AIは便利だが、固有名詞や数字など重要な部分は人がチェックするのが原則

手順

  1. 修正したい資料ファイルがある場所で Claude Code を起動する。可能なら元ファイルをコピーして控えを残す。
  2. どこをどう直したいかを具体的に伝える。例：「この資料の中の『Cloud Code』をすべて『Claude Code』に直してください」「見出しの大きさをそろえてください」。
  3. Claude Code が提案する修正内容を確認する。意図と違えば「箇条書きはそのままで、本文だけ直して」のように言葉で調整する。
  4. 修正を反映してもらい、変更された箇所を自分の目で確認する。固有名詞・数字・日付は特に念入りに見る。
  5. 同じ直しを今後もくり返すなら、「この修正ルールをスキルにして」と頼み、次回から呼び出せるようにしておく。

💡 補足・コツ

  • 「どう直すか」をできるだけ具体的に伝えるほど、思い通りの結果になります。あいまいな指示よりも実例を1つ示すのが効果的です。
  • 大きく直したいときは、一度に全部ではなく「まず見出しだけ」「次に本文」のように段階的に頼むと、確認しながら安全に進められます。
4

コンテンツ生成を自動化してみよう

ここまでは「整理・修正」でしたが、最後は新しく作り出す自動化です。複数の画像をまとめて生成する流れを通して、AIに制作作業を任せるイメージをつかみましょう。

  • 1枚ずつ手作業で作るのではなく、作りたい内容のリストを渡して一括で生成すれば、同じ作業を何度もくり返す手間がなくなる
  • たとえば「商品名と説明の一覧表」から、各商品ぶんの画像をまとめて作る、といった『リスト → 大量生成』の流れが組める
  • ポイントはテイスト（色味・サイズ・雰囲気）を指示でそろえること。条件を最初に決めておくと、何枚作っても統一感のある仕上がりになる
  • うまくいった生成の手順はスキルとして登録しておくと、次からは新しいリストを渡すだけで同じ品質のコンテンツを量産できる
  • 画像生成には専用のAIサービス（画像生成API）が必要になる場合がある。利用には利用規約と権利関係の確認を忘れずに、安心して使える範囲で活用する

手順

  1. 生成したいコンテンツの一覧（リスト）を用意する。例：商品名・説明・希望する雰囲気を並べた表やメモ。
  2. 仕上がりの条件をそろえて伝える。例：「すべて正方形・同じ色調で、明るく清潔感のある雰囲気にしてください」。
  3. Claude Code に「このリストの内容に沿って、画像をまとめて生成してください」と依頼する。必要なサービスやキーの準備はその場で案内してもらう。
  4. できあがったコンテンツを確認し、気になる点は「もう少し明るく」「背景は白に」など言葉で調整して作り直してもらう。
  5. 満足できる流れができたら「この生成の手順をスキルにして」と頼み、次回からはリストを差し替えるだけで再利用できるようにする。

💡 補足・コツ

  • 最初は1〜2枚だけ試し、テイストが思い通りか確認してから本数を増やすと、無駄なやり直しを減らせます。
  • 生成したコンテンツを仕事で使う場合は、画像生成サービスの利用規約や商用利用の可否を必ず確認しましょう。
