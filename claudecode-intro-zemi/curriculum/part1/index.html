Claude Code入門ゼミ ／ 教材構成 PART ①

導入

このパートでは、Claude Code が「そもそも何なのか」「何ができて、いくらで、どう始めるのか」を、コードを書いたことがない方でも迷わないように一つずつ確認していきます。手を動かす前に全体像をつかみ、安全に・楽しく使い始めるための土台をつくるのがゴールです。

1

コースの全体像とゴール

最初に、このコースで何を学び、どこまでできるようになるのかを共有します。ゴールが見えていると、途中でつまずいても「今どこにいるか」がわかり、安心して進められます。

  • 対象は完全な初学者・非エンジニア。プログラミングの前提知識がなくても、日本語の指示だけで成果物を作れるようになることを目指します。
  • 学習の流れは「導入で全体像と使い方をつかむ → 実際にWebサイトやアプリを作る → 業務効率化に応用する」という順番。やさしい題材から少しずつ難易度を上げていきます。
  • このコースを最後まで進めると、会社のホームページや簡単なAIアプリ、日々の作業の自動化あたりまで、自分の手で作れるイメージが持てるようになります。
  • 本コースは「手取り足取りのハンズオン」ではなく自習を前提とした学習の場です。動画と資料を見ながら、自分のペースで何度も見返して進めてOKです。
  • まずはイメージを持つことが大切。すべてを完璧に理解しようとせず、「こういうことができるんだ」という感覚を先に育てましょう。

💡 補足・コツ

  • わからない用語が出てきても、その場で止まらず先に進んでOK。多くは後の回で具体例とともに腑に落ちます。
  • 重要な操作やコツは、コース内で何度も繰り返し出てきます。一度で覚えられなくても問題ありません。
2

Claude Code とは

ここでは「そもそも Claude とは何か」から始めて、Claude Code が普段のチャットAIと何が違うのかを整理します。名前に「Code」と付いていますが、エンジニア専用のツールではありません。

  • Claude（クロード）は、アメリカの Anthropic（アンソロピック） 社が開発したAI。ChatGPT のように、質問すると賢く答えてくれる対話型AIです。
  • Claude Codeは、その Claude を土台にしたAIエージェント。質問に答えるだけでなく、あなたのパソコン上でファイルを作ったり書き換えたり、指示を自分で何ステップも実行してくれます。
  • チャット版の Claude / ChatGPT との一番の違いは「手を動かしてくれるか」。チャットAIは相談相手（壁打ち）、Claude Code は実際に作業して成果物を仕上げてくれる部下、とイメージするとわかりやすいです。
  • Claude のデスクトップアプリには Chat（会話）／Cowork（一緒に作業）／Code（開発・自動化）のタブがあります。本コースで主に使うのは、自由度がもっとも高い Code（＝Claude Code） です。
  • 似たツールに OpenAI 社の Codex などもあります。それぞれ得意分野が違いますが、本コースでは情報量が多く拡張性に優れた Claude Code を使って学んでいきます。

💡 補足・コツ

  • 「Anthropic って何？」と身構えなくて大丈夫。Claude を作っているアメリカの会社、とだけ覚えておけば十分です。
  • 横文字が苦手でも安心してください。Claude Code を起動して「〇〇ってなに？」と日本語で聞けば、あなたのレベルに合わせて説明してくれます。
3

Claude Code でできること

Claude Code は「コードを書く」ことに留まらず、ビジネスや日常の作業まで幅広くこなせます。言葉での説明だけでなく、実際に動く様子を見て「こんなことができるんだ」という具体的なイメージを持ちましょう。

  • 制作・開発：会社のホームページ、Webアプリ、スマホアプリなどを、日本語の指示だけで作れます。既存ファイルの修正や整理（リファクタリング）も得意です。
  • 業務効率化：書類やレポートの自動作成、ファイルの整理・リネーム、請求書や領収書のPDF作成など、面倒な事務作業を任せられます。
  • 調べもの・分析：Web検索と組み合わせて、競合調査やリサーチを代行する「AI秘書」のような使い方もできます。
  • ポイントは、パソコン上でできることの多くを、自然な日本語で頼めること。プログラミングを何年もかけて学ばないとできなかったことが、グッと身近になります。
  • デモを見ると一番伝わります。実際に指示を出し、ファイルが自動で生成される様子を見ることで、抽象的だったイメージが一気に具体的になります。

💡 補足・コツ

  • 「自分の仕事の、どの面倒な作業を任せたいか」を1つ思い浮かべながら見ると、活用例が自分ごとになります。
  • 今日や今週「面倒だな」と感じた作業を3つ書き出してみると、Claude Code で効率化できそうなことが見つけやすくなります。
4

料金プランについて

Claude Code を自分のパソコンで動かすには、Claude の有料プランが必要です。ここでは本コースで前提とするプランと、料金の考え方を確認します。お金の話は最初に整理しておくと安心です。

  • 本コースでは Pro プラン を前提に進めます。月額の定額プランで、決まった範囲内で Claude Code を使えるのが特徴です。
  • 料金には大きく2タイプあります。定額（サブスク）＝月いくらで使い放題に近い形と、従量課金＝使った分だけ支払う API の形。本コースの基本は定額の Pro プランです。
  • サブスクプランには一定時間ごとに利用枠がリセットされる仕組みがあります。たくさん試して枠を使い切っても、時間が経てばまた使えるので、失敗を恐れず練習できます。
  • 「まず動画でイメージを持ってから検討」でもOK。手元で動かすには課金が必要ですが、どうしても今は難しい場合は、先に内容を見て「何ができるか」をつかんでから契約を考えても問題ありません。
  • 料金体系や提供プランは変わることがあります。契約前に Claude（Anthropic）の公式サイトで最新の内容を必ず確認してください。

💡 補足・コツ

  • 従量課金（API）は便利な反面、使った分だけ費用が発生します。初学者のうちは料金が読みやすい定額プランから始めるのがおすすめです。
  • 「壊しても大丈夫」な練習用フォルダを作っておくと、心置きなく試行錯誤できます。
5

アカウント作成と機能紹介

実際に使い始めるための、アカウント作成からプラン契約までの流れを確認し、画面をひととおり見ていきます。最初の登録さえ済めば、あとは指示を出すだけです。

  • 大まかな流れは、① アカウント登録 → ② プラン（Pro）契約 → ③ 画面のクイックツアー。順番どおりに進めれば迷いません。
  • 登録にはメールアドレスなどが必要です。普段使っているアドレスで問題ありません。
  • プラン契約の画面では、定額の Pro プラン のほかに、使った分だけ支払う従量課金（API）の選択肢も出てきます。本コースでは定額の Pro を選びます。
  • 契約後は、Claude の画面構成（Chat / Cowork / Code など）をざっと見て、「どこに何があるか」の地図を頭に入れておきましょう。
  • 細かな設定は今はすべて覚えなくて大丈夫。「ログインして、プランを選び、起動できる」状態になればこの段階はクリアです。

手順

  1. Claude（Anthropic）の公式サイトにアクセスし、メールアドレスなどでアカウントを新規登録する。
  2. プラン選択画面で、本コースで使う Pro プラン を選んで契約する。
  3. 契約が完了したら、画面のタブ（Chat / Cowork / Code など）をひととおり開いて、全体の配置を確認する。
  4. 準備ができたら、次の回で実際に Claude Code を起動して、簡単な指示を出してみる。

💡 補足・コツ

  • 登録時のメールアドレスとパスワードは、後で困らないようにメモしておきましょう。
  • 画面の見た目は更新で変わることがあります。ボタンの位置が動画と少し違っても、表示名や役割を手がかりに落ち着いて探せば見つかります。
6

ビジネス利用における注意点

Claude Code は強力なぶん、使い方を誤ると情報漏えいや事故につながります。安心して使い続けるために、最初に「やってはいけないこと」と「設定しておくべきこと」を押さえましょう。ここは特に大切な回です。

  • 学習設定をオフにする：入力した内容がAIの学習に使われない設定が用意されている場合があります。業務や機密の情報を扱うなら、まず学習設定をオフにしておくと安心です。
  • 機密情報の扱いに注意：顧客情報・個人情報・パスワード・社外秘の資料などは、安易に入力しない。会社のルールがある場合は、それに必ず従ってください。
  • 何でもPC操作を任せない：Claude Code はファイルの作成・変更・削除までできます。大事なファイルがある場所で無防備に作業させると、意図しない変更が起きるおそれがあります。
  • 大事な操作は「承認しながら」進める：実行前に確認を求められる場面では、内容を読んでから許可しましょう。すべて自動で進める設定は、慣れて安全性を理解してから検討すれば十分です。
  • 練習と本番を分ける：壊れても困らない練習用フォルダを用意し、重要なデータと切り離して試すのが、事故を防ぐいちばん簡単な方法です。

💡 補足・コツ

  • 迷ったら「これは外部に出してはいけない情報か？」と一度立ち止まって考える習慣をつけましょう。
  • API キーやパスワードなどの秘密情報は、本人以外（AIを含む）に渡さない・画面に貼らないのが基本です。
7

Claude Code の使い方

いよいよ Claude Code を起動して、簡単な指示を出してみます。基本の操作はとてもシンプルで、覚えることはごくわずか。まずは「起動して、頼んで、結果を見る」の流れを体験しましょう。

  • 起動はコマンドを1つ打つだけ。あとは普段の日本語で「〇〇して」と頼むだけで動き出します。難しいコマンドを暗記する必要はありません。
  • 指示を出すと、Claude Code が自分で考えて手順を実行し、必要なファイルを作ったり調べたりしてくれます。途中で許可を求められたら、内容を確認して進めます。
  • 権限（パーミッション）の確認が出たら、その操作を許可するか選びます。よく使う操作は「常に許可」を選ぶと、毎回聞かれずスムーズになります。
  • 終了したいときは Ctrl+C を2回押します（1回では止まらない仕様です）。誤操作で作業が中断しないための安全策です。
  • 最初の一歩としては、たとえば「あるテーマを調べて、要点を1つのファイルにまとめて」のような小さな指示を試すのがおすすめです。

手順

  1. 作業用のフォルダを用意し、そのフォルダで Claude Code を起動する。
  2. やってほしいことを日本語で1つ頼む（例：「〇〇について調べて、要点をまとめたファイルを作って」）。
  3. 実行中に許可を求められたら、内容を確認して許可する（よく使う操作は「常に許可」でもOK）。
  4. できあがった結果（生成されたファイルなど）を確認する。
  5. 終わったら Ctrl+C を2回押して終了する。

💡 補足・コツ

  • 指示は具体的なほど良い結果になります。「何を」「どんな形で」出してほしいかを一言添えましょう。
  • 思いどおりにならなくても、続けて「ここをこう直して」と頼めば修正してくれます。やり直しは何度でもできます。
8

バイブコーディングの基本

「バイブコーディング」とは、コードを自分で書く代わりに、AIに自然な言葉で指示しながら作っていく進め方のことです。ここでは、つまずいたときやエラーが出たときにどうするか、その対処の基本を身につけます。困ったら、まずは Claude に聞く——これが合言葉です。

  • 基本姿勢は「小さく始める」。最初から完璧な大作を狙わず、シンプルなものを作って少しずつ育てていくと、挫折しにくくなります。
  • エラーは怖くない。うまく動かないときは、エラーメッセージをそのまま Claude Code に貼り付けて「これを直して」と頼むのが基本。多くは1〜2回のやり取りで直ります。
  • 指示は具体的に。「動かない」だけでなく、「どの画面で」「何をしたら」「どうなった／どうしたい」を添えると、的確に直してくれます。画面のスクリーンショットを添えるのも有効です。
  • 困ったらまず Claude に聞く。用語の意味、次にやるべきこと、エラーの原因——わからないことは遠慮なく Claude Code 自身に質問しましょう。あなた専用の先生がそばにいる感覚です。
  • 失敗を恐れない。サブスクの利用枠は時間が経てばリセットされるので、何度でも作り直せます。試行錯誤こそが上達の近道です。

手順

  1. 作りたいものを小さく言葉にして、Claude Code に頼む。
  2. 出てきた結果を確認し、直したい点があれば「ここをこうして」と続けて指示する。
  3. エラーが出たら、エラーメッセージをそのまま貼り付けて「修正して」と頼む（必要なら画面のスクショも添える）。
  4. 意味がわからない言葉や手順は、その場で「〇〇ってなに？」「次は何をすればいい？」と Claude Code に質問する。
  5. うまくいくまで②〜④を繰り返す。行き詰まったら、この回を見返して対処の型を思い出す。

💡 補足・コツ

  • 「正常に動くように、すべての機能が動作するように直して」のように、ゴールを言葉で示すと修正してもらいやすくなります。
  • この回は困ったときのマニュアルとして使えます。今後エラーや不明点が出たら、まずここに戻ってきてください。
