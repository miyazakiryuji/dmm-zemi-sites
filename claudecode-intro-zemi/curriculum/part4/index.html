Web開発②：漫画制作アプリをつくる（OpenAI API連携）

このパートでは、入力したセリフやシーンから漫画風の画像をつくる「漫画制作アプリ」を、Claude Code と一緒に開発します。まず見た目だけの試作品（モックアップ）を作り、そこに OpenAI の API をつないで実際に動かし、最後は自分だけがアクセスできる形でこっそりWeb公開するところまでを体験します。コードは書けなくて大丈夫です。「何を作りたいか」を日本語で伝えるのがあなたの仕事です。

1

漫画制作アプリの動作イメージ

まずは完成形を見ます。「これから自分はこんなものを作るんだ」というゴールが頭にあると、途中の作業がぐっと分かりやすくなります。

  • 完成した漫画制作アプリは、ユーザーがセリフや場面の説明を入力すると、それをもとに漫画風のコマ（画像）を生成して画面に表示する、というのが基本の動きです。
  • 裏側では、入力された文章を OpenAI の API に送り、返ってきた画像を画面に並べる、という流れが動いています。仕組みの詳細は後のトピックで扱うので、今は「入力 → AIが生成 → 表示」の3ステップだけ覚えれば十分です。
  • 完成デモを見るときは、細かいコードではなく「画面のどこに何があるか」「ボタンを押すと何が起きるか」に注目してください。これがそのまま、Claude Code に伝える指示の材料になります。
  • 「自分ならこういう機能が欲しい」と感じたら、メモしておきましょう。後でアプリをブラッシュアップする際の改善アイデアになります。
  • 完璧な作品を作るのが目的ではありません。「AIと対話するだけでアプリが形になる」体験そのものが、このパートのいちばんの学びです。
2

漫画制作アプリ開発の全体像

いきなり完成品を作ろうとせず、小さなステップに分けて進めます。全体の地図を先に持っておくと、今どこにいるのか迷わなくなります。

  • 開発は大きく「①見た目だけの試作品（モックアップ）を作る → ②OpenAI API をつないで本物の生成を動かす → ③使い勝手を磨く → ④自分だけ見られる形で公開する」という順番で進みます。
  • 最初から API をつながないのには理由があります。まず見た目を固めてから中身（AI処理）を入れると、問題が起きたときに「見た目の問題」か「API の問題」かを切り分けやすく、つまずきにくくなります。
  • それぞれの工程で、あなたがやることは「Claude Code に日本語で要望を伝える」「動かして確認する」「気になった点を再度伝える」の繰り返しです。コードを直接書く必要はありません。
  • API をつなぐ工程では料金とセキュリティの話が出てきます（トピック5・8・10）。お金が絡む部分なので、ここだけは少していねいに進めます。
  • 全体を通して、一度に完璧を目指さず「動く小さな状態」を積み上げるのがコツです。Claude Code は何度でも修正に付き合ってくれます。

手順

  1. 完成イメージ（ゴール）を確認する
  2. API連携前のモックアップを作る
  3. OpenAI API の契約と設定を済ませる
  4. モックアップに OpenAI API を連携して実際に動かす
  5. 動作と使い勝手をブラッシュアップする
  6. 自分だけがアクセスできる形でWeb公開する
3

使用する技術の説明

このアプリで登場する技術を、ごく簡単に言葉だけ押さえておきます。仕組みを丸暗記する必要はなく、「こういう役割の道具がある」とイメージできればOKです。

  • API（エーピーアイ）とは、外部のサービスの機能を「呼び出して使わせてもらう窓口」のことです。今回は OpenAI が用意した画像生成・文章生成の機能を、自分のアプリから呼び出して使います。
  • OpenAI APIは、ChatGPT を提供している OpenAI 社が公開している窓口です。自分のアプリから文章や画像を生成してもらえます。利用には後述のAPIキー（合言葉のような鍵）が必要です。
  • APIキーは、あなた専用のパスワードのようなものです。これを使った分だけ料金が発生するため、他人に絶対に見せない・コードに直書きしないのが鉄則です（詳しくはトピック5・10で扱います）。
  • アプリの画面そのものは、Claude Code が HTML / CSS / JavaScript といったWebの基本技術や、開発を助けるフレームワークを使って組み立ててくれます。あなたが用語を覚える必要はありません。
  • 用語が難しく感じても止まらなくて大丈夫です。分からない言葉が出てきたら、その場で Claude Code に「〜って何？初心者向けに教えて」と聞くのがいちばんの近道です。
4

モックアップを開発してみよう

まずは API をつなぐ前の「見た目だけの試作品（モックアップ）」を Claude Code に作ってもらいます。中身はまだ空っぽでも、画面の形が見えると一気に開発が進みます。

  • モックアップとは、本物の機能はまだ入っていない「見た目とボタンだけのお試し版」です。家を建てる前の間取り図のようなもので、完成像を全員で確認するために作ります。
  • Claude Code には「どんな画面が欲しいか」を具体的に伝えるのがポイントです。たとえば「セリフを入力する欄」「シーンを説明する欄」「生成ボタン」「結果を並べる場所」のように要素を挙げると、意図通りの画面になりやすいです。
  • この段階では、生成ボタンを押しても仮の画像やサンプル文が出る状態でかまいません。API はまだつないでいないので、本物のAI生成は次以降のトピックで行います。
  • 出来上がった画面を見て「ボタンの位置を変えたい」「入力欄を増やしたい」と思ったら、そのまま日本語で伝えれば直してくれます。気軽に何度も注文を出すのがコツです。
  • ローカルサーバーを起動すると、自分のパソコンのブラウザで試作品を確認できます。起動コマンドが分からなければ Claude Code に「このアプリを起動して確認させて」と頼めばOKです。

手順

  1. Claude Code に「漫画制作アプリのモックアップを作りたい」と伝える
  2. 欲しい画面の要素（入力欄・ボタン・結果表示エリアなど）を具体的に列挙する
  3. ローカルサーバーを起動してブラウザで見た目を確認する
  4. 気になった点を日本語で伝えて修正してもらう
  5. 画面の形に納得できたら、次の API 連携へ進む

💡 補足・コツ

  • うまく伝わらないときは、参考にしたいアプリやサイトのイメージを言葉で説明すると精度が上がります。
  • 完璧を目指さず「だいたいこの形」まで来たら先に進みましょう。後からいくらでも調整できます。
5

OpenAI API の契約と設定

ここからは本物のAI生成を動かすための準備です。OpenAI のアカウントを作り、APIキーを発行し、使いすぎを防ぐ上限設定までを行います。お金が関わる工程なので、ていねいに進めましょう。

  • OpenAI API は基本的に従量課金（使った分だけ支払う）です。生成のたびにわずかな料金がかかるため、最初に支払い情報の登録と利用上限の設定をセットで済ませておくと安心です。
  • APIキーは発行時にしか全体を表示できないことが多いので、表示されたら安全な場所に控えておきます。万一見失っても、新しいキーを発行し直せば大丈夫です。
  • 最大の注意点はAPIキーの漏洩です。キーは「あなたの財布の鍵」と同じで、他人に渡ると勝手に使われ、その分の料金があなたに請求されます。画面共有・スクショ・チャット貼り付け・SNS投稿で映り込ませないよう徹底してください。
  • 使いすぎへの備えとして、OpenAI の管理画面で月ごとの利用上限（リミット）を設定できます。学習用なら低めの上限にしておくと、想定外の高額請求を防げます。
  • もしキーをうっかり外部に出してしまったら、ためらわず即座にそのキーを無効化（削除）して、新しいキーを発行します。「たぶん大丈夫」と放置しないことが、いちばんのセキュリティ対策です。

手順

  1. OpenAI のアカウントを作成し、ログインする
  2. 支払い情報を登録する（従量課金のため）
  3. 月ごとの利用上限（リミット）を学習用に低めで設定する
  4. APIキーを発行し、表示された鍵を安全な場所に控える
  5. 発行したキーはコードに直書きせず、後の連携工程で安全な方法で渡す（トピック6・10参照）

💡 補足・コツ

  • APIキーは sk- のような決まった形式で始まる長い文字列です。コピーするときは前後に余計な空白や改行が混ざらないよう注意しましょう。
  • 利用上限の設定は「保険」です。少額でも設定しておくと、初めての従量課金でも落ち着いて学べます。
6

モックアップに OpenAI API を連携してみよう

いよいよ試作品に「本物の頭脳」を入れます。発行したAPIキーを安全に渡し、Claude Code に連携処理を実装してもらって、実際に漫画が生成される瞬間を目指します。

  • APIキーはコードに直接書き込まず、.env という専用の設定ファイルに入れて読み込ませるのが基本です。こうすると、キーをコード本体やGitHubに載せずに済み、漏洩リスクを大きく下げられます。
  • Claude Code には「モックアップに OpenAI API を連携して、入力内容から実際に画像を生成するようにして」のように依頼します。どのファイルを直すかまで自分で把握する必要はありません。
  • 生成ボタンを押すと、入力した内容が OpenAI に送られ、返ってきた結果が画面に表示されます。仮の画像が本物の生成結果に置き換わったら連携成功です。
  • エラーが出ても慌てないでください。エラーメッセージをそのまま Claude Code に貼り付けて「これを直して」と頼むのが、いちばん早い解決法です。料金不足・キーの誤り・モデル指定ミスなどはよくあるつまずきです。
  • 生成には数秒〜数十秒かかることがあります。ボタンを連打するとその回数だけ料金がかかるので、結果が出るまで落ち着いて待ちましょう。

手順

  1. プロジェクト内に .env ファイルを用意し、APIキーを記入する
  2. Claude Code に「.env のキーを使って OpenAI API を連携して」と依頼する
  3. ローカルで起動し、実際に入力して生成ボタンを押す
  4. 本物の生成結果が表示されるか確認する
  5. エラーが出たらメッセージをそのまま Claude Code に渡して直してもらう

💡 補足・コツ

  • うまく動かないときは「キーは正しいか」「利用上限や残高は足りているか」「.env を正しく読めているか」の3点をまず疑いましょう。
  • .env は次のトピック以降で .gitignore によりGitHubから除外します。今のうちから「キーは .env、公開しない」とセットで覚えておきましょう。
7

モックアップをブラッシュアップしてみよう

基本の生成が動いたら、使い勝手や見た目を磨いていきます。ここで Claude Code を効率よく動かすための便利機能（CLAUDE.md や Skills）も活用します。

  • ブラッシュアップは「機能追加」と「使いやすさ改善」の両面で進めます。「複数コマをまとめて生成」「生成中の待機表示を出す」「結果を保存できるようにする」など、欲しい動きを日本語で伝えていきます。
  • CLAUDE.mdは、Claude Code への「業務マニュアル」のようなファイルです。プロジェクトのルールやよく使う指示を書いておくと、毎回説明し直さなくても意図通りに動いてくれます（詳しくはトピック8）。
  • Skills（スキル）は Claude Code の機能を拡張する仕組みです。たとえばfind-skills（必要なスキルを探してくれるスキル）やskill-creator（スキル自体を作るスキル）を使うと、デザイン改善やコードレビューなどを得意な道具に任せられます。
  • 改善は一度に1つずつ頼むのがコツです。まとめて大量に注文すると、どこで問題が起きたか分かりにくくなります。小さく直して、動作確認して、また次へ、を繰り返しましょう。
  • 「もっと良くしたい」と思ったら、完成済みのアプリでも遠慮なく注文を続けて構いません。対話しながら育てていけるのが、Claude Code を使った開発のいちばんの強みです。

手順

  1. 改善したい点を1つに絞って Claude Code に伝える
  2. 修正後、ローカルで動かして結果を確認する
  3. 必要に応じて find-skills で便利なスキルを探し、導入する
  4. 繰り返し使う指示やルールは CLAUDE.md にまとめておく
  5. 満足できる仕上がりになるまで、小さな改善を積み重ねる

💡 補足・コツ

  • スキルを入れても呼び出せないときは、Claude Code をいったん終了して再起動すると認識されることがあります。
  • 改善案が思いつかないときは「初心者が使いやすくなる改善を提案して」と聞くと、Claude Code がアイデアを出してくれます。
8

Claude Code の応用設定（CLAUDE.md・Skills・Rules）

ここでは Claude Code をもっと賢く・安全に使うための3つの仕組みを整理します。特に Rules は、これからGitHubなど外部サービスを触る前に入れておきたいセキュリティの土台になります。

  • CLAUDE.mdは、プロジェクトごとのルールを書いておくファイルです。「コメントは日本語で」「.env は触らない」「ファイル削除前に必ず確認する」のような決まりを書くと、Claude Code が毎回それに従ってくれます。
  • 中身を自分でゼロから書くのが難しければ、/init コマンドを使うと、プロジェクトを自動で分析して CLAUDE.md のたたき台を作ってくれます。最初は空のままでも問題ありません。
  • Skillsは、特定の作業が得意な「拡張パーツ」です。デザイン改善・セキュリティチェック・コードレビューなど、目的に合ったスキルを find-skills で探したり、skill-creator で自作したりできます。
  • Rules（ルール設定）は、Claude Code に「やってはいけないこと」を前もって定めておく安全装置です。GitHubなど外部サービスを操作し始める前に、危険な操作を勝手に実行させないテンプレ的な設定を入れておくと安心です。
  • これらはすべて「意図通りに・安全に動いてもらうための準備」です。設定は一度入れれば以降のセッションでも効くので、面倒に感じても最初に整えておく価値があります。

手順

  1. /init でCLAUDE.mdのたたき台を作る、または必要なルールを手書きで足す
  2. 「.env は触らない」「削除前に確認」など、安全に関わるルールを CLAUDE.md に明記する
  3. find-skills で目的に合うスキルを探し、必要なものを導入する
  4. GitHubなど外部を触る前に、危険操作を防ぐ Rules を設定しておく
  5. 設定後は Claude Code を再起動して反映を確認する

💡 補足・コツ

  • CLAUDE.md のファイル名は大文字小文字まで正確に。claude.md のように小文字だと読み込まれません。
  • 「何を禁止すべきか分からない」ときは、Claude Code に「初心者が安全に使うための推奨ルールを提案して」と聞くのが手早いです。
9

サイトにアクセス制限をかける方法

作ったアプリを「自分だけ」「決まった人だけ」が使える状態にする方法を学びます。全世界に公開する前に、まずは限定的に使えるようにするのが安全です。

  • アクセス制限が役立つ場面はいくつもあります。毎回ローカルサーバーを起動するのが面倒/スマホや外出先からも使いたい/まずは限られた人にだけテスト公開したい——こうしたときに「限定公開」が便利です。
  • 代表的な制限方法は2つあります。①認証（パスワードやログインを要求する）と②IP制限（特定のネットワークからのアクセスだけ許可する）です。それぞれ仕組みが違うので、用途に合わせて選びます。
  • 認証は「合言葉を知っている人だけ入れる」イメージです。どこからアクセスしても、正しいパスワードやログインができれば使えるので、外出先やスマホからの利用に向いています。
  • IP制限は「この場所（ネットワーク）からだけ入れる」イメージです。自宅や会社など決まった回線からのアクセスに限定でき、それ以外は弾けます。
  • OpenAI API を使うアプリは、誰でもアクセスできる状態にすると他人にAPIを使われ、その分の料金があなたに請求される恐れがあります。だからこそ、公開前のアクセス制限はお金を守るうえでも重要です。

💡 補足・コツ

  • 「自分だけ使えればいい」なら、認証とIP制限を組み合わせると安心感が高まります（次のトピックで実践します）。
  • 設定方法が分からなければ、用途を伝えて Claude Code に「このアプリに認証をかけたい」「特定の人だけ使えるようにしたい」と相談しましょう。
10

漫画制作アプリをウェブに限定公開してみよう

最後に、二重のアクセス制限（認証＋IP制限）をかけたうえで、作った漫画制作アプリをWebに限定公開します。APIキーを扱うアプリの公開なので、機密情報の取り扱いがいちばんの肝です。

  • 公開前の最重要チェックはAPIキーを絶対に外部へ出さないことです。キーは .env に入れ、その .env を .gitignore に書いてGitHubへアップロードされないよう除外します。これを忘れるとキーが世界中に公開されてしまいます。
  • 万一キーがGitHubなどに載ってしまったら、流出した前提で即座にそのキーを無効化し、新しいキーを発行します。一度ネットに出たキーは「もう漏れたもの」として扱うのが鉄則です。
  • 限定公開では認証（パスワード等）とIP制限を二重にかけることで、「自分や許可した人だけが使える」状態を作ります。これにより、勝手にAPIを使われて料金が膨らむリスクを大きく減らせます。
  • 公開作業そのものは Claude Code に手伝ってもらえます。「.env と .gitignore の設定を確認して、APIキーが公開されない状態でデプロイしたい」のように、安全面を明示して依頼するのがポイントです。
  • 公開できたら、必ず自分以外からはアクセスできないか/APIキーがソースコードに含まれていないかを確認しましょう。「動いた」だけで満足せず、「安全に動いているか」までチェックして完成です。

手順

  1. APIキーが .env に入っていることを確認する
  2. .gitignore に .env を記載し、GitHubへ載らないよう除外する
  3. 認証（パスワード等）とIP制限の二重のアクセス制限を設定する
  4. Claude Code に「APIキーが公開されない状態でデプロイして」と依頼する
  5. 公開後、許可した人以外がアクセスできないこと・キーが漏れていないことを確認する

💡 補足・コツ

  • 公開前に Skills のセキュリティチェック系スキルを回すと、見落としやすい漏洩リスクを洗い出せます。
  • 心配なときは、テスト用に低めの利用上限を設定したままにしておくと、万一の際も被害を最小限に抑えられます。
