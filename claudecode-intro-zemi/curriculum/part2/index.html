Web制作 ― Claude Codeで会社ホームページを作って公開する

このパートでは、プログラミングの知識がなくても、Claude Code との会話だけで会社ホームページを一から作り、インターネットに公開するまでを体験します。ファイルの役割をざっくり理解するところから始め、制作・デザイン調整・公開へと、ひとつずつ手を動かしながら進めましょう。

1

ホームページ制作の基礎知識

会社ホームページが、どんな部品（ファイル）の組み合わせでできているのかを、ざっくりイメージできるようにします。まずは最終的に作るサイトの完成形を見てから、その中身を分解して役割を覚えましょう。

  • 最初に「これから作る会社ホームページ」の完成形を眺めると、ゴールが見えて学びやすい。難しい文法は覚えず、各部品の役割をつかむのが目的です
  • Webページは主に HTML・CSS・JavaScript の3種類のファイルでできています。家に例えると、HTML＝骨組み（柱・土台）、CSS＝見た目（壁や屋根の色・形）、JavaScript＝動き（押すと反応するスイッチ）
  • HTML は「何を表示するか」を決める部分。見出し・段落・画像・リンクなどを <h1> や <p> といった「タグ」で囲んで意味づけします。CSSなしのHTMLだけだと、装飾のない黒い文字が縦に並ぶだけの素朴な見た目になります
  • CSS は「見た目」を整える部分。色・文字の大きさ・余白・配置などを指定します。同じHTMLでもCSSを当てると、一気にきれいなホームページらしい見た目に変わります
  • JavaScript は「動き」を担当。スマホで開いたときのメニュー開閉や、ボタンを押したときの反応など、クリックに応じた動作を作ります。静的なホームページなら必須ではありませんが、あると便利です

💡 補足・コツ

  • 細かい書き方は一切覚えなくて大丈夫です。コードはすべて Claude Code が書いてくれます。皆さんは「色を変えたい→CSS」「動きを変えたい→JavaScript」という役割の対応だけ頭に入れておけば、修正の指示が的確に出せます
  • HTMLとCSSは「マークアップ言語」と呼ばれ、計算や条件分岐をするプログラミング言語（JavaScriptなど）とは少し性質が違います。区別がつかなくなったら Claude Code に「これはHTML・CSS・JavaScriptのどれの担当？」と聞けば即答してくれます
2

会社ホームページを制作してみよう

いよいよ Claude Code に会社ホームページを作ってもらいます。最初はごく簡単な指示だけで、たった数分で土台となるサイトが出来上がる体験をしてみましょう。

  • まずサイト用の新しいフォルダを1つ用意します。フォルダ名は company-homepage のように英数字にしておくと、あとで公開するときにトラブルが起きにくくなります
  • Claude Code には専門用語を使わず、自然な日本語で頼んでOK。たとえば「Claude Codeを使ったホームページ制作の会社を始めます。HTML・CSS・JavaScriptの3つのファイルで、会社ホームページを作ってください」程度で十分です
  • Claude Code が index.html（トップページ）をはじめ、見た目用の style.css、動き用の script.js などのファイルを自動で作ってくれます。承認を求められたら許可して進めます
  • 出来上がったファイルは、ブラウザで開けばそのまま見た目を確認できます。フォルダ内の index.html をダブルクリック、またはエディタから開くだけです
  • スマホ表示の確認は、ブラウザの開発者ツールを使います。Windowsは F12、Macは Cmd + Option + I で開き、画面上部のスマホ表示アイコンに切り替えると、メニュー開閉などの動きも試せます

手順

  1. サイト専用の新しいフォルダ（例: company-homepage）を英数字の名前で作る
  2. そのフォルダをエディタで開き、Claude Code を起動する
  3. 「HTML・CSS・JavaScriptの3ファイルで会社ホームページを作って」と自然な日本語で依頼する
  4. ファイル作成や編集の確認を求められたら許可して、生成が終わるまで数分待つ
  5. 出来上がった index.html をブラウザで開き、見た目を確認する
  6. 開発者ツール（Win: F12 / Mac: Cmd + Option + I）でスマホ表示や動きもチェックする

💡 補足・コツ

  • 指示を送ったつもりが途中で改行できずに送信されてしまうことがあります。改行は Shift + Enter または Alt + Enter、送信は Enter だけ、と覚えておくと長い指示も書きやすくなります
  • もしCSSやJavaScriptが作られず素朴な見た目のままだったら、「見た目を整えるCSSと、動きをつけるJavaScriptのファイルも作ってください」と追加で頼めば補ってくれます
  • この段階の見た目は「とりあえず形になった」レベルで構いません。デザインを整えるのは次のトピックで行います
3

会社ホームページをブラッシュアップしてみよう

最初に出てきたサイトは、どこか「AIが作った感」のある素朴なデザインになりがちです。ここでは具体的な指示を重ねて、自分の会社らしい洗練された見た目へと肉付けしていきます。

  • デザインの雰囲気を言葉で伝えるのが一番手軽です。「ミニマルな感じで」「上品で落ち着いた印象で」「ポップで親しみやすく」など、イメージを伝えるだけで方向性が変わります。用語が思いつかなければ「このサイトに合うデザインの方向性をいくつか提案して」と聞けばOK
  • 内容（コンテンツ）を増やす指示も有効です。「会社概要・サービス紹介・料金プラン・よくある質問・お問い合わせのセクションを追加して」のように、欲しい項目を具体的に挙げると一気に充実します
  • 細部の調整もお願いできます。「日本語が読みやすいフォントにして」「マウスを乗せたときに軽くアニメーションする程度の動きを、派手すぎない範囲で加えて」など、ピンポイントの注文に応えてくれます
  • 複数のデザイン案を見比べたいときは、「ミニマル案とカード分割（ベントーグリッド）案を、ボタンで切り替えて比較できるようにして」と頼めば、同じページ上で見くらべて選べます
  • 採用するデザインが決まったら、「比較用のボタンは消して、ミニマル案を正式採用してください」と伝えて仕上げます

💡 補足・コツ

  • より洗練されたデザインを最初から狙うなら、フロントエンドデザイン用の Skill を活用する方法があります。Skill は Claude Code に専門的な作法を覚えさせる仕組みで、これを使うと「AIっぽさ」の少ない、モダンで完成度の高い見た目を生成しやすくなります（Skill の詳しい仕組みは別のパートで扱います）
  • 他社サイトを参考にするときは、そっくりそのまま真似るのは著作権上NGです。色味・余白・全体の構成といった「雰囲気」だけを参考にしましょう。参考にしたい見た目はスクリーンショットを貼ると伝わりやすいです
  • 気に入った配色やフォントの設定は、メモ（Markdownファイル）に残しておくと、次に別のサイトを作るときに「前回のこの設定で」と再利用できて便利です
4

ホームページ公開の基礎知識

作ったサイトは、そのままでは自分のパソコンの中でしか見られません。世界中の人に見てもらうには「公開（デプロイ）」が必要です。その前に、ドメインとサーバーという2つの言葉をやさしく整理しておきます。

  • パソコンの中だけにあるサイトは、自分しか見られない「下書き」状態です。誰でもアクセスできるようにする作業を公開（デプロイ）と呼びます
  • サーバーは、サイトのファイルを置いておき、いつでも訪問者に届けてくれる「土地・置き場所」のようなものです。インターネット上に常に立っていて、アクセスがあるとページを返してくれます
  • ドメインは、そのサイトの「住所」にあたる文字列です（例: example.com）。訪問者はこの住所をブラウザに入力することで、サーバー上のサイトにたどり着けます
  • 昔は「レンタルサーバーを契約して、専用ソフトでファイルを送り込む」という手間のかかる方法が主流でした。費用もかかり、設定も初心者には難しいものでした
  • 今はホスティングサービスを使えば、ファイルをアップロードするだけで、サーバーの用意も住所の発行も自動でやってくれます。初学者はまずこの簡単な方法から始めるのがおすすめです

💡 補足・コツ

  • 最初は専用ドメイン（独自の住所）を買う必要はありません。ホスティングサービスが あなたのサイト名.pages.dev のような無料の住所を自動で用意してくれるので、それで十分に公開できます
  • 独自ドメインは、サイトを本格運用したくなってから後付けで設定できます。まずは無料の住所で「公開できた」という体験を優先しましょう
5

会社ホームページをウェブで公開してみよう

いよいよ作ったサイトをインターネットに公開します。ここでは Cloudflare Pages という無料のホスティングサービスを使い、ファイルをドラッグ＆ドロップするだけの一番かんたんな方法で公開します。

  • Cloudflare Pages は、静的なホームページを無料で公開できるサービスです。難しいサーバー契約や専用ソフトは不要で、ファイルをアップロードするだけで世界中からアクセスできるURLが発行されます
  • 公開には Cloudflare の無料アカウントが必要です。メールアドレスで登録でき、料金プランの入力なしで始められます
  • ダッシュボードから新しいプロジェクトを作り、「ファイルをドラッグ＆ドロップ（Direct Upload）」を選択。サイトのフォルダ（または ZIP にまとめたもの）をそのまま枠に放り込んでアップロードします
  • アップロードが終わって公開すると、あなたのプロジェクト名.pages.dev という形のURLが自動で発行されます。このURLを共有すれば、誰でもサイトを見られます
  • サイトを直したくなったら、Claude Code でファイルを修正し、同じプロジェクトに新しいファイルを再アップロードするだけで更新が反映されます

手順

  1. 公開したいサイトのフォルダの中に、トップページが必ず index.html という名前で入っているか確認する
  2. Cloudflare の無料アカウントを作成し、ログインする
  3. ダッシュボードで「Pages（Workers & Pages）」から新規プロジェクトを作成し、ドラッグ＆ドロップ方式を選ぶ
  4. プロジェクト名を入力し、サイトのフォルダ（または ZIP）を枠にドラッグ＆ドロップしてアップロードする
  5. アップロード完了後「保存して公開（Save and Deploy）」を実行し、発行された ～.pages.dev のURLにアクセスして表示を確認する
  6. 内容を修正したら、Claude Code で直したファイルを同じプロジェクトに再アップロードして更新する

💡 補足・コツ

  • トップページのファイル名は必ず index.html に。top.html や homepage.html だと、URLを開いても「ページが見つかりません」になってしまいます
  • 公開後にデザインが崩れたり画像が出ない場合、ファイルの場所の指定（パス）がパソコン内の特定の場所を指していることが原因のことが多いです。Claude Code に「公開先でも正しく表示されるよう、ファイルの参照を相対パスに直して」と頼めば直せます
  • お問い合わせフォームは「見た目」だけだと送信してもメールは届きません。実際に受け取れるようにするには、フォーム送信を受け付ける別の仕組み（フォーム用サービスや送信先の設定）が必要になります。まずは見た目だけ用意しておき、必要になったら Claude Code に相談しながら追加するのがおすすめです
