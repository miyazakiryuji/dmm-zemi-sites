Claude Code入門ゼミ ／ はじめての基礎

はじめに知っておきたい 3つの基礎

Claude Code をさわる前に、「Claude とは」「Claude Code とは」「Web版・デスクトップ版・CLI版の違い」の3点を、完全な初学者向けにやさしく整理しました。

Claude とはClaude Code とはWeb版・デスクトップ版・CLI版の違い

POINT 1

Claude とは

Anthropic がつくった対話型の生成AI（＝“頭脳”そのもの）

まず Claude（クロード） は、AI企業 Anthropic（アンソロピック） が開発した生成AI（大規模言語モデル）です。日本語で話しかけると、文章づくり・要約・翻訳・調べ物・コード生成などをこなします。ChatGPT や Gemini と並ぶ、代表的な生成AIのひとつです。

Claude をひとことで言うと

Claude は、アメリカ・カリフォルニア州に拠点を置く AI 企業 Anthropic が開発した対話型の生成AIです。ChatGPT（OpenAI）の代表的なライバルとして知られています。

私たちが日本語で質問や指示を入力すると、Claude がその意味をくみ取り、文章・要約・翻訳・アイデア出し・プログラムのコードなどを生成して返してくれます。

  • 開発元：Anthropic（アンソロピック）— 米国カリフォルニア州の AI 企業
  • 種類：対話型（チャット型）の生成AI／大規模言語モデル（LLM）
  • 入力：日本語をふくむ自然言語。専門知識がなくても話し言葉で使える
  • 得意なこと：文章生成・要約・翻訳・調査の壁打ち・コード生成 など

💡 用語ミニ解説

  • 生成AI：文章や画像などを“生成”する AI の総称。
  • LLM（大規模言語モデル）：大量の文章で学習し、言葉を扱うのが得意な AI の仕組み。
  • Anthropic：Claude をつくっている会社。安全性を重視した AI 開発で知られる。

Claude の特徴

Claude は特に、長い文章やコードをまとめて読み取り・整理することや、コード生成の品質で高い評価を受けています。また Anthropic は「AI を安全で誠実にふるまわせる」ことを重視した開発方針を掲げています。

  • 長文に強い：長い資料やコードをまとめて読み、要約・整理するのが得意
  • コードが得意：プログラム生成の品質に定評があり、開発支援で広く使われる
  • 安全性を重視：危険・不誠実な出力を避ける方針で設計されている

モデルには“種類”と“バージョン”がある

Claude には用途別に複数の“モデル”があり、性能と速度のバランスが異なります。バージョンは頻繁に更新されるため、最新の名称は公式サイトで確認してください。

  • Opus（オーパス）：最高性能。むずかしい・大きなタスク向け
  • Sonnet（ソネット）：性能と速度のバランス型。日常使いの主力
  • Haiku（ハイク）：軽量・高速。手早い処理向け

ℹ️ モデルのバージョンは数か月単位で新しくなります（例：Opus 4 系 など）。本資料では特定の版に依存しないよう、系統名（Opus / Sonnet / Haiku）で説明しています。

Claude を使う“入口”はいくつもある

同じ Claude（頭脳）を、いろいろな“入口”から使えます。

  • Web版（claude.ai）：ブラウザで開いてチャット。いちばん手軽
  • デスクトップアプリ：Mac / Windows のアプリ。スマホアプリもある
  • Claude Code（CLI）：ターミナルで動く“エージェント”。本ゼミのメイン（次の項目）
  • API：開発者が自分のアプリに Claude を組み込むための接続口

💡 ポイント：「Claude」は頭脳そのもの。それを どの入口（Web／アプリ／ターミナル）で使うか が、3番目の項目で扱う“版の違い”です。

POINT 2

Claude Code とは

Claude をターミナルで動かし、ファイル操作まで自動でこなす“AIエージェント”

Claude Code は、Anthropic が開発した AIエージェント です。ターミナル（黒い画面）で日本語の指示を出すと、Claude が自分でコードを書き、ファイルを操作し、複数の作業を自動で進めてくれます。名前に “Code” と付きますが、コーディング専用ではなく、調査・資料作成・業務自動化まで幅広く使えます。

Claude Code をひとことで言うと

Claude Code は Anthropic が開発した AIコーディングエージェント です。ターミナルという“コマンドを打つ黒い画面”の上で動きます。

「このホームページを作って」「このアプリを作って」と 日本語で指示を出すだけ で、AI が自動でコードを書いたり、パソコン上のファイルを操作したりしてくれます。

Claude Code というのはコーディングだけじゃなくて、ビジネスのあらゆる場面で使える AI エージェントになります。— DMM 生成AI CAMP『Claude Code とは』(EP02) より

なぜ“ターミナル”で動くの？

ターミナルは、ファイルの操作やコマンドの実行を、文字（テキスト）で行える環境です。AI が「何をするか」を文字でそのまま指示・確認できるため、AI が直接作業を進めやすい のです。

「黒い画面」と聞くと身構えますが、入門で実際に打つのは claude や node -v 程度。コマンドを暗記する必要はありません。

💡 ChatGPT などが“ブラウザの中で会話する”のに対し、Claude Code は“あなたのパソコンの中で実際に手を動かす”イメージです。

ChatGPT / Gemini との違い

観点ChatGPT / GeminiClaude Code
動作場所ブラウザターミナル
PC内のファイル操作アップロードした内容は読めるが、PC内のファイルは操作しないパソコン上のファイルを直接操作できる
自律実行一問一答連続実行・複数ステップを自動
得意分野質問対応・文章生成開発・業務の自動化

出典：DMM 生成AI CAMP『Claude Code とは』(EP02) の比較表をもとに整理

できること①：コーディング・開発

メインはやはり“ものづくり”。日本語の指示から、こんなものを作れます。

  • Webサイト・ホームページ：会社HP やランディングページ など
  • Webアプリ・スマホアプリ：入力フォームや診断ツール、簡単なアプリ など
  • 既存コードの修正：バグ修正・リファクタリング・テストコードの作成

ℹ️ リファクタリング＝ごちゃごちゃしたコードを整理して読みやすくすること。テストコード＝プログラムが正しく動くか確かめるためのコード。（EP02 の用語解説より）

できること②：コーディング以外の仕事

“Code” と付きますが、非エンジニアの業務にも使えます。EP02 では次のような用途が紹介されています。

  • プロジェクト管理：進捗・スケジュール・タスクの管理
  • 新規事業・リサーチ：競合調査、ビジネスモデルの検討
  • 調査・分析・戦略立案：Web 検索を自動で実行して情報を集める
  • AI 秘書 / 事務：リサーチ代行、請求書・領収書などの PDF 発行

“新人の部下”にたとえると

Claude Code は AIエージェント＝「指示すると自律的に複数ステップを実行する AI」です。EP03 ではこれを“新人の部下”にたとえています。

新人がいきなり自己流で仕事を始めたら不安ですよね。Claude Code は、あらかじめ ルールや業務マニュアルを渡しておける のが強みです。

  • CLAUDE.md：プロジェクトの“ルールブック”。守ってほしい約束を書いておくファイル
  • Skills：作業手順を“マニュアル化”して、決まった仕事を任せられる仕組み
  • サブエージェント / Agent Teams：複数の AI を並行で動かし、大きな作業を分担できる

💡 自由度の高さ（CLAUDE.md・Skills で細かく制御できる点）が、Cursor や Codex と比べたときの Claude Code の独自の強みです。（EP03）

どうやって使い始める？

導入の詳細は後の回で手を動かしながら進めますが、流れはシンプルです。

はじめ方の流れ

  1. Node.js を入れる（準備）
  2. ターミナルで npm install -g @anthropic-ai/claude-code を実行してインストール
  3. プロジェクトのフォルダで claude と打って起動
  4. あとは日本語で「○○を作って」と指示するだけ
  • 本ゼミでは、この後の回で実際のインストールと操作を、手を動かしながら進めます（Node.js＋npm 方式：EP07／EP08）。
  • なお現在はネイティブインストーラでの導入方法もあり、推奨手順は更新されることがあるため、最新は公式サイトでご確認ください。

POINT 3

Web版・デスクトップ版・CLI版の違い

同じ Claude を“どの入口で使うか”。手軽さ重視か、作る力重視か

Claude（頭脳）は同じでも、使う“入口（容れ物）”によってできることが変わります。ここでは代表的な 3 つ — Web版・デスクトップアプリ・CLI版（＝Claude Code） — の違いを整理します。結論から言うと、気軽に試すなら Web、普段使い＋ツール連携ならデスクトップ、実際に作る・自動化するなら CLI（本ゼミのメイン） です。

① Web版（claude.ai）

ブラウザで開くだけ。いちばん手軽な入口

  • 使い方：ブラウザで claude.ai を開いてチャットするだけ。インストール不要
  • できること：質問・文章作成・要約・翻訳、ファイルや画像をアップロードして読ませる、Web 検索 など
  • PC内のファイル：アップロードしたものは読めるが、あなたの PC 内のファイルを自由に読み書きはしない（ブラウザの中で完結）
  • 料金：無料枠あり。有料プラン（Pro など）で利用量が増える
  • 向いている人：まず生成AI を気軽に体験したい人／調べ物・文章作成が中心の人

ℹ️ 最近は claude.ai 上から、ブラウザ／クラウドで動く Claude Code（Claude Code on the web）も使えるようになってきています。提供機能は更新されるため、最新は公式で確認してください。

② デスクトップアプリ（Mac / Windows）

Web と同じ体験をネイティブアプリで。外部ツール連携に強い

  • 使い方：Mac / Windows にアプリをインストールして常駐。起動が速く OS と馴染む（スマホアプリ版もある）
  • できること：基本は Web版と同じチャット。加えて MCP に対応し、外部のツールやデータと連携できる
  • PC内のファイル：MCP 連携を設定すれば、対応ツール経由で手元のデータを扱える
  • 向いている人：毎日 Claude を使う人／対応ツールと連携して使い込みたい人

💡 MCP（Model Context Protocol）＝AI と外部ツール／データをつなぐ“共通の差し込み口”。対応サービスを足すと、できることが広がります。

③ CLI版 ＝ Claude Code（本ゼミのメイン）

ターミナルで動く“エージェント”。実際に手を動かして作る・自動化する

  • 使い方：ターミナルにインストールし、claude で起動。プロジェクトのフォルダ上で動かす
  • できること：ローカルのファイルを直接読み書き、コマンドを実行、複数ステップを 自律的に 進める
  • 制御の仕組み：CLAUDE.md・Skills・サブエージェントで、ルールや手順を渡して動かせる
  • 向いている人：Web サイト・アプリを作りたい人／作業を自動化したい人（＝本ゼミの内容）

ℹ️ Claude Code は CLI（ターミナル）が基本ですが、いまは デスクトップアプリ・Web（claude.ai/code）・IDE拡張（VS Code / JetBrains） からも使えます。

💡 迷ったら（結論）

  • とりあえず試す → Web版（claude.ai）をブラウザで開く
  • 毎日使う・ツール連携 → デスクトップアプリ
  • 実際に作る・自動化する → CLI版＝Claude Code（本ゼミで扱うのはこれ）
  • どれも“頭脳は同じ Claude”。入口を変えても、覚えた感覚はそのまま活かせます。

FAQ

よくある質問

つまずきやすいポイントをまとめました。

「Claude」と「Claude Code」は何が違うの？
Claude は頭脳（AI そのもの）、Claude Code はその Claude をターミナルで動かし、ファイル操作や自動実行までさせる“道具（エージェント）” です。Web版でチャットするのが Claude、ターミナルで手を動かさせるのが Claude Code、と整理すると分かりやすいです。
非エンジニアでも Claude Code を使える？
はい。指示は 日本語の話し言葉でOK。入門で打つコマンドも claude 程度で、暗記は不要です。EP02 でも非エンジニア向けの用途（リサーチ・PDF 発行・プロジェクト管理など）が紹介されています。
ChatGPT と何が違うの？
ChatGPT / Gemini は基本 ブラウザでの“一問一答・壁打ち”。Claude Code は ターミナルであなたの PC のファイルを操作し、複数ステップを自動 で進めます。“質問の相手”と“手を動かす部下”の違いです。
無料で使える？
Claude 自体は 無料枠 から試せます。利用量や使える機能はプランによって変わり、料金は更新されるため、最新は公式で確認してください。
どれから始めればいい？
まず Web版（claude.ai） で生成AI の感触をつかみ、本ゼミの本番である CLI版（Claude Code） に進むのがおすすめです。
