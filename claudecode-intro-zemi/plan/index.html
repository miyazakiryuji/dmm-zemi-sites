📝 米メモ
  • アプリ開発から、指示送信後に何度も確認を求められるので Auto Mode を紹介 → でも入門者向けに最初から Auto Mode を紹介すると、Claude Codeのリスクについての考えが甘くなるかもしれない。入門コースだからこそ「承認をしながら順次進めるのが安全」ということを、何度も承認しながら体験してもらったほうがよく、Auto Mode の紹介は二本目から、なんなら付録として最後に紹介するでもよさそう
  • （Planモードは、コースのどこかで必ず触れたい。ここで使うのがよいとも思ったが、一本目のアプリ開発で受講生が動画と同じ手順で進められず、同じようなアプリが作れず問い合わせに繋がる可能性もあるため、Planモードの紹介は最後の業務効率化や付録あたりでよいかも）

  • （例）「以下のエラーメッセージが表示されたので、修正して」 → エラー画面をキャプチャして添付するとか

3

秘書アプリにデザインを当ててみよう

想定4分

秘書アプリにデザインを当てる指示

💬 松井回答
  • ルーレットアプリより診断メーカーの方が確かに魅力的。ただ Web開発① はAPIを使わないシンプルな題材にしたい。ルーレットアプリより良い題材があるはずなので、いったん保留としたい。
  • 論点「所詮はAPIを連携するだけ」について：個人的には、ここがどうしても初心者にはハードル高いのでは、と当初思ったものの、APIでつまずく箇所は概ね想定でき（API設定ミスがほとんど）、かつエラーが出ても今のモデルは優秀なため1〜2回の指示で修正されるはず。そう考えると Web開発① で「秘書アプリ」を採用するのもありかもしれない。ワークショップで扱われた題材を確認し、難易度を当講座に合わせてアレンジする方向で検討。
  • 前提として、入門者にとってAPI連携が「簡単」だとは全く思わないが、それを理由に漫画制作アプリを諦めるほどの難易度か？というと疑問。「APIの概念を自分で調べて理解し、自分で調べながらAPIキーを取得・設定して、実行時のエラーを自力で解決」だとかなり難しいが、すべてレクチャーするわけなので、ほぼクリアできそう。（n8nコースでもAPI設定＆呼び出しをしているが、そこまで難易度高いかな？と思う）
  • マスターコースのAIチューターの質問履歴に、APIキーの無料枠超過に関する質問が2件あったが、APIキー関連の問い合わせは見た覚えがない。APIキーの取得・設定は基本的に動画の手順通りに進めればいいだけでは、ということを考えると、漫画制作アプリを諦めるほどの難易度でもない気がする。「手順通りにAPIキーを取得＆設定して、APIキーが何かはよく分からないが、とりあえず動画の手順通りに動かしてみる」ぐらいはできそうで、そこまでいければ正確かつ深い理解はいらないのでOKな気もする（APIキーが何かよくわからず日常的に使っている非エンジニアも結構いそう）
