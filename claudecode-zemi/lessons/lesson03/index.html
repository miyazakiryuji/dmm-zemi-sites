Key Points

今回押さえる3つのポイント

LIVE講義でも重点的に解説する観点です。自習時の視聴メモの目安にもどうぞ。

1

CLAUDE.md は Claude Code への“業務ルール”。毎回自動で読み込まれる

プロジェクトのルートに置くと、Claude Code が起動するたびに自動で読み込みます。「コメントは日本語で」「本番のデータベースは触らない」のような単純なルールからでOK。何を書くか迷ったら最初は空のままでも問題なく、/init でプロジェクトを分析して自動生成もできます。

2

Skills は“特定タスク専用”の専門スキル。CLAUDE.md と使い分ける

CLAUDE.md＝常に守る全体ルール（プロジェクトに1つ）、Skills＝その時だけ使う専門知識・手順書（複数OK）。.claude/skills/<名前>/SKILL.md に置き、自動起動かスラッシュコマンド（例 /frontend-design）で呼び出します。講座では基本スラッシュコマンドで呼びます。

3

HTML / CSS / JS は役割だけ。コードは Claude Code が書く

HTML＝骨格・CSS＝見た目・JavaScript＝動き。文法は覚えなくて大丈夫で、修正したい時に「CSSのフォントを変えて」のように指示できれば十分です。会社HPは「要件を伝える → 生成 → ブラウザで確認」で2〜3分で形になります。まずは“ポン出し”で1枚作ってみるのが今回のゴール。

When You're Stuck

分からないことが出てきたらこの順番で

学習中・課題実施中に詰まったら、以下のフローで進めてください。1人で抱え込まないことが一番大事です。今回は座学（CLAUDE.md／Skills）に加えて初めてのHP制作。ファイルのパスや改行（Alt+Enter）まわりで詰まりやすいので、気軽に投げてください。

1STEP 1

まずは
スクショしてAIに聞く

エラー画面・つまずいた画面を スクリーンショット で撮り、Claude／ChatGPT／Gemini など普段使っている生成AIに貼り付けて質問してみてください。

精度を上げるコツ：
  • 何をしようとしていたか
  • どうなる予定だったか
  • 実際にどうなったか
をスクショに添えて投げる。

2STEP 2

それでも解決しなければ
ゼミ掲示板へ

AIに聞いても解決しない／そもそも何を聞けばいいか分からない場合は、DMM 生成AI CAMP 学び放題 の上部ナビゲーション 「ゼミ掲示板」 から投稿してください。他の受講生・運営・講師が回答できます。

投稿テンプレ：
  • 実現したかったこと
  • 試したこと（STEP 1 のAIとのやり取りも含めるとなお良い）
  • つまずきポイント（スクショ歓迎）
  • 環境（OS／ツール／バージョン）

「これは聞いていいレベルか…」と迷う必要はありません。STEP 1 → STEP 2 の順番で気軽に出してください。
