Key Points

今回押さえる3つのポイント

LIVE講義でも重点的に解説する観点です。自習時の視聴メモの目安にもどうぞ。

1

会社HPは「仕上げ」と「公開」でひとまず完成させる

「ミニマル」「ニューモーフィズム」「ベントーグリッド」のようなデザイン用語で指示すると狙ったテイストを出しやすく、複数案を生成してボタンで切り替え比較できます。日本語フォント・マイクロアニメーション・ホバーまで足して“リッチ化”し、最後は Netlify Drop にドラッグ＆ドロップするだけで公開できます。

2

生成AIアプリは Python / Streamlit / API の3つの役割でできている

Python＝処理を書く言語、Streamlit＝UIとロジックを同時に作る道具、API＝外部のAI（Claude等）を呼び出す窓口。この3層が組み合わさって「入力→生成AIが処理→画面に表示」というアプリになります。文法は覚えなくてOK、コードは Claude Code が書きます。

3

まずは Python を入れて、動くツールを1つ作ってみる

実践②の最初の山はPythonのインストール（Windows は「パスを通す」設定に注意）。越えたら、作りたい機能を自然言語で伝え、シンプルな構成を指定 → 生成 → ローカルで Streamlit を起動してブラウザ確認、という流れで“動く生成AIツール”が手元にできます。

When You're Stuck

分からないことが出てきたらこの順番で

学習中・課題実施中に詰まったら、以下のフローで進めてください。1人で抱え込まないことが一番大事です。今回はHPの公開や Python のインストールなど環境まわりで詰まりやすいので、気軽に投げてください。

1STEP 1

まずは
スクショしてAIに聞く

エラー画面・つまずいた画面を スクリーンショット で撮り、Claude／ChatGPT／Gemini など普段使っている生成AIに貼り付けて質問してみてください。

精度を上げるコツ：
  • 何をしようとしていたか
  • どうなる予定だったか
  • 実際にどうなったか
をスクショに添えて投げる。

2STEP 2

それでも解決しなければ
ゼミ掲示板へ

AIに聞いても解決しない／そもそも何を聞けばいいか分からない場合は、DMM 生成AI CAMP 学び放題 の上部ナビゲーション 「ゼミ掲示板」 から投稿してください。他の受講生・運営・講師が回答できます。

投稿テンプレ：
  • 実現したかったこと
  • 試したこと（STEP 1 のAIとのやり取りも含めるとなお良い）
  • つまずきポイント（スクショ歓迎）
  • 環境（OS／ツール／バージョン）

「これは聞いていいレベルか…」と迷う必要はありません。STEP 1 → STEP 2 の順番で気軽に出してください。
